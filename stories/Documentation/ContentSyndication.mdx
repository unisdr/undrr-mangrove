# Content syndication

Mangrove supports embedding syndicated content from the UNDRR platform into any page. Content blocks pull live data from UNDRR's Elasticsearch backend and render as styled card grids -- news, publications, events, and other content types.

The `SyndicationSearchWidget` component powers syndication embeds when configured without the search interface. A typical syndicated content block uses a fixed query, hides the search box and filters, and renders results as cards:

```html
<div data-mg-search-widget
  data-default-query="disaster risk reduction"
  data-display-mode="card"
  data-grid-columns="3"
  data-results-per-page="6"
  data-show-search-box="false"
  data-show-facets="false"
  data-show-results-count="false"
  data-require-image="true">
</div>
```

The `Fetcher` component provides a lower-level data-fetching utility for custom rendering of UNDRR API data.

## Accessibility

Syndicated content blocks load dynamically, so assistive technology support requires extra care:

- Card grids use semantic list markup (`<ul>` / `<li>`) so screen readers can convey the number of results and allow list navigation.
- Images include alt text sourced from the CMS. The `requireImage` filter controls whether results without images are shown -- it does not mark images as decorative.
- Loading states announce to assistive technology via `aria-live` regions so users are informed when content is being fetched.
