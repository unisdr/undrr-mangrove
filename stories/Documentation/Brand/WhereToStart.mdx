# Where to start

The Brand section is a layered guide. New readers can pick up any page, but the pages are easier to use when read in order. This page is the canonical reading path.

If you are short on time, the **Five-minute path** at the bottom gives you the irreducible core.

---

## Reading order

1. **<LinkTo kind="brand-about-this-guide" story="docs">About this guide</LinkTo>** — a one-page overview of UNDRR's web properties with theme cards. Start here if you want to see the sub-brands at a glance.

2. **<LinkTo kind="brand-brand-guidelines" story="docs">Brand guidelines</LinkTo>** — the editorial and visual identity baseline. Brand positioning (Knowledgeable, Approachable, Collaborative), communication rules, logo usage, photography standards, web elements, typography. This is the foundation that everything else is built on.

3. **<LinkTo kind="brand-layered-identity" story="docs">Layered identity</LinkTo>** — the strategic model behind UNDRR's identity: core UNDRR → sub-brand → Regional Office (Layer 2.5, regional expression of core) → campaign / partner. Read this before any of the practical pages below — it gives you the framing they all share. Includes a worked IDDRR campaign example and notes on dual-parented campaigns (Global Platform editions).

4. **<LinkTo kind="brand-visual-voice" story="docs">Visual voice</LinkTo>** — colour, typography, photography, iconography, layout rhythm, motion, illustration, data visualisation. Read in parallel with written voice; both pages share the same K / A / C anchor.

5. **<LinkTo kind="brand-written-voice" story="docs">Written voice</LinkTo>** — the four registers (Urgent, Authoritative, Warm, Hopeful), house conventions (British English, sentence case, em-dash discipline, descriptive links), and a tone-rewrite reference table.

6. **<LinkTo kind="brand-audience-mapping" story="docs">Audience mapping</LinkTo>** — how voice and visual emphasis shift across UNDRR's five primary audiences (governments, practitioners, local governments, private sector, media and citizens), with a worked one-story-five-framings example.

7. **<LinkTo kind="brand-sub-brand-rationale" story="docs">Sub-brand rationale</LinkTo>** — why each sub-brand (PreventionWeb, MCR2030, IRP, ARISE, DELTA Resilience) looks and sounds different from the others, and what stays constant across all of them.

8. **<LinkTo kind="brand-anti-patterns" story="docs">Anti-patterns and pre-publish checklist</LinkTo>** — a shortlist of anti-patterns worth naming explicitly, plus the unified pre-publish checklist (Must hold / Should hold / Craft) that every other page links to.

9. **<LinkTo kind="brand-ai-imagery-policy" story="docs">AI imagery policy</LinkTo>** — the 12 standing rules for generative imagery (no photoreal humans, no children, no rendered text, named country, disability-as-peer, decolonial framing, hex colours, etc.) plus the institutional sources. Read before generating anything.

10. **<LinkTo kind="brand-ai-imagery-prompts" story="docs">AI imagery prompts</LinkTo>** — three holistic example prompts, the UNDRR prompt formula (your subject + style block + output spec), and an LLM meta-prompt you can paste into ChatGPT or Claude with a rough brief and get back an on-brand image prompt. Also: language-model prompts for alt text, headlines, tone register, and British-English conversion.

11. **<LinkTo kind="brand-brand-identity" story="docs">Brand identity</LinkTo>** — interactive theme-aware page showing live colour, typography, logo, button, and icon previews per sub-brand. Use the theme picker in the Storybook toolbar to switch between properties.

12. **<LinkTo kind="brand-common-patterns" story="docs">Common patterns</LinkTo>** — plain-language reference for foundations shared across all themes (breakpoints, grid classes, accessibility standards, language support, icon sources, z-index).

13. **<LinkTo kind="brand-component-gallery" story="docs">Component gallery</LinkTo>** — decision guidance for the moments when multiple components could plausibly do the job (which hero, which card, which pagination, etc.). Sits on top of the Storybook sidebar (which is the full catalog); does not re-list every component.

## Skip-to paths

### If you are a designer

- Layered identity → Visual voice → Sub-brand rationale → Anti-patterns → Brand identity (interactive) → Component gallery.

### If you are a writer or content editor

- Brand guidelines → Written voice → Audience mapping → Anti-patterns → AI imagery prompts (the language-model section, plus the LLM meta-prompt if you are generating imagery).

### If you are a site editor producing AI imagery

- AI imagery policy → AI imagery prompts (use the formula, or paste your rough brief into the LLM meta-prompt) → Anti-patterns checklist (run every output through Must-hold rows before publishing).

### If you are a developer

- About this guide → Layered identity → Common patterns → Component gallery → Brand identity (for live previews of the theme tokens).

### If you are an external partner or content producer

- Brand guidelines → Layered identity → Sub-brand rationale (for whichever sub-brand you are producing for) → Anti-patterns → AI imagery policy (for the constraint set).

## Five-minute path

If you only have five minutes:

1. **The voice trio.** Read "Brand positioning" in <LinkTo kind="brand-brand-guidelines" story="docs">brand guidelines</LinkTo>. UNDRR is Knowledgeable, Approachable, Collaborative. Pressure-test every choice against these three.
2. **The layered model.** Skim the diagram and "What threads through every layer" in <LinkTo kind="brand-layered-identity" story="docs">layered identity</LinkTo>. Core UNDRR threads through every sub-brand and every campaign.
3. **The anti-pattern shortlist.** Skim the "must hold" section of the <LinkTo kind="brand-anti-patterns" story="docs">anti-patterns</LinkTo> checklist. These six items prevent the most common drift.

After five minutes you have the irreducible core. The rest of the guide deepens these three.

## A note on how this guide evolves

The Brand section is a living guide. Voice registers, sub-brand rationale, AI imagery prompts, and anti-patterns are written from current understanding and evolve as UNDRR Communications updates institutional guidance and as we learn more from sub-brand owners and content producers. If you find a gap or a contradiction, file an issue or open a pull request on the [Mangrove repository](https://github.com/unisdr/undrr-mangrove). The most consequential corrections come from the people who use the guide every day.
