# Visual voice

Visual voice is how the brand sounds when you cannot read the words. Colour, type, photography, iconography, layout rhythm, motion, illustration, and data viz all carry the same three characteristics — **Knowledgeable, Approachable, Collaborative** — and connect every artifact back to UNDRR's <LinkTo kind="brand-layered-identity" story="docs">layered identity</LinkTo>.

This page is the practical companion to the <LinkTo kind="brand-brand-guidelines" story="docs">brand guidelines</LinkTo>. The guidelines say *what* to do; this page says *how it should feel*.

---

## How each visual medium carries voice

One row per medium. The K/A/C column names the characteristic this medium leans on hardest. The locked column is the thread that does not flex per sub-brand or campaign; the variable column is what may flex (and where).

| Medium | What it signals | K/A/C anchor | Locked thread | Variable | Avoid |
| --- | --- | --- | --- | --- | --- |
| **Colour** | Brand recognition; state and hierarchy | All three | Corporate Blue as the institutional anchor; Sendai stripe in lockup contexts; neutrals core-locked | Sub-brand interactive primary (PreventionWeb teal, MCR2030 purple, IRP bright blue, ARISE burgundy, DELTA dark navy); one campaign accent on top of sub-brand | Decorative-only colour; rainbow chart palettes; Sendai stripe as wallpaper |
| **Typography** | UN-family credibility before anything else loads | Knowledgeable | Roboto + Roboto Condensed (UNDRR-adopted web typefaces, aligned with UN visual identity practice — "UN official" attribution pending confirmation with `undrrcomms@un.org`); sentence case; Arabic via `:lang(ar)` → Noto Kufi Arabic + Dubai | Heading scale per layout; weight per emphasis | Theme-specific font swaps; American-English-shaped headings; em-dashes in UI |
| **Photography** | "Approachable" and "Collaborative" made visible | A + C | Real people, named, in their context; credit and source on every photo; positive prevention/recovery framing | Hero crop and aspect per use; sub-brand subject emphasis (researchers for PreventionWeb, municipal staff for MCR2030, etc.) | Conference rooms; decorative-only stock; disaster-tourism framing; hopeless aftermath |
| **Iconography** | DRR concepts (OCHA) vs web UI (Mangrove) | Approachable | OCHA humanitarian icons for hazards; Mangrove icon set for web UI | None per-theme — same icon set, same controls | Font Awesome on new work; mixed icon systems in one control row |
| **Layout rhythm** | Reading rhythm that makes sub-brand articles feel like cousins | Approachable | Spacing scale (`_variables.scss`); 60–75 char line length; vertical rhythm | None per-theme — spacing is core-locked | Theme-specific spacing hacks; one-off spacing values inside components |
| **Motion** | Functional state change; never decoration | Approachable | 150ms hover, 200ms reveal; respect `prefers-reduced-motion` | Campaign motifs may have micro-interactions; sub-brand chrome may not | Decorative motion; animation that breaks the layer above |
| **Illustration** | Place-rooted story, not decoration | A + C | Editorial illustration for explainers; diagrammatic for systems; human-centred when used | Per-campaign motif; per-sub-brand subject emphasis | AI-generic isometrics; "blue gradient NGO" aesthetic; uncanny copy-paste figures |
| **Data viz** | Sourced rigour over persuasion | Knowledgeable | Sendai reduced-saturation palette for sequential / qualitative scales; Corporate Blue for primary series; source citation on every chart | Sub-brand primary may replace Corporate Blue for the primary series in sub-brand contexts | Rainbow palettes; decorative outlines; marketing-flavoured chart framing |

For the canonical "what threads through every layer / what flexes at each layer" table, see <LinkTo kind="brand-layered-identity" story="docs">layered identity</LinkTo>. For the full font table including web fallbacks and the Arabic override note, see the typography section of <LinkTo kind="brand-brand-guidelines" story="docs">brand guidelines</LinkTo>. For the photo size table, see <LinkTo kind="brand-brand-guidelines" story="docs">brand guidelines</LinkTo>. > **Tip:** When in doubt about whether a visual choice belongs in a layout, ask whether it serves *meaning* (state, hierarchy, brand) or *decoration*. UNDRR is institutionally serious; decorative-only choices tend to read as generic.

## Photography (in depth)

Photography is the single strongest expression of UNDRR's "Approachable" and "Collaborative" characteristics. Everything else flows from getting this right — which is why it gets its own section rather than living only in the table above.

<HighlightBox>
  <strong>SharePoint Photography & Icons page:</strong> "Conference pictures should be kept to the strict minimum (we do not want to convey that image)."
</HighlightBox>

| Do | Avoid |
| --- | --- |
| Show what people are *doing* — clearing, rebuilding, planning, monitoring | The disaster itself; aftermath without agency |
| Real people, named, in their context — community-led, not crew-shots | Decorative-only stock with no place, no name, no story |
| Display credit and source on every photo (policy, not preference) | Conference rooms; suited delegates; plenary halls |
| Positive prevention and recovery framing | Disaster-tourism imagery; hopeless framing |

For generative imagery (AI illustration when a real photo is not available), see the <LinkTo kind="brand-ai-imagery-policy" story="docs">AI imagery policy</LinkTo> and the <LinkTo kind="brand-ai-imagery-prompts" story="docs">prompt cookbook</LinkTo>.

## Motion (in depth)

Motion is a quiet thread today, but the order of these principles matters — read them in sequence:

1. **Function over delight.** Motion confirms an action or signals a state change. It is rarely decorative.
2. **Respect `prefers-reduced-motion`.** UNDRR audiences include disabled users; motion that ignores the system preference fails the brand's "Approachable" characteristic. This is non-negotiable.
3. **Never animate at the campaign layer in a way that breaks the sub-brand layer.** A campaign motif can have its own micro-interaction; the sub-brand chrome should not.

If principle 2 conflicts with principle 1 (e.g. a state-confirmation animation a reduced-motion user cannot see), provide a non-motion equivalent — a colour change, an icon swap, a focus state shift. Never silently skip the confirmation.
