Mangrove provides shared content structures for UNDRR web properties: cards, heroes, navigation, footers, and syndicated content blocks. Each property keeps its own brand colors, component treatments, and application-specific interfaces. The content patterns are shared; your visual identity, application logic, and user experience flows stay yours. Mangrove is used on [undrr.org](https://www.undrr.org), [preventionweb.net](https://www.preventionweb.net), [mcr2030.undrr.org](https://mcr2030.undrr.org), and partner sites.

Each UNDRR property themes components independently. The shared layer is the content structure -- the parts that let syndicated content render correctly and accessibility fixes propagate to every site at once.

Use the theme switcher in the toolbar above to preview components in UNDRR, PreventionWeb, IRP, or MCR2030 styles. For more on what Mangrove covers, what it doesn't, and how it fits alongside other frameworks, see <LinkTo kind="getting-started-about-mangrove" story="docs">About Mangrove</LinkTo>.

## Structure

Mangrove uses a loose [atomic design](https://atomicdesign.bradfrost.com/chapter-2/) approach. The Storybook sidebar is organized as:

- **Getting started**: What Mangrove is, onboarding guides for different integration approaches
- **Design decisions**: Colors, spacing, typography, and foundational design elements
- **Components**: Cards, buttons, forms, navigation, and interactive elements
- **Utilities**: CSS utilities, loaders, and layout helpers

To request a new component or report an issue, [open a GitHub issue](https://github.com/unisdr/undrr-mangrove/issues). [Internal project notes are on the GitLab Wiki](https://git.un.org/undrr/web-backlog/-/wikis/Mangrove:-the-UNDRR-Component-library).
