UNDRR Mangrove is a component library for consistent, accessible UNDRR web development.

**<LinkTo kind="getting-started-mangrove-the-undrr-standard" story="docs">Mangrove is the UNDRR standard</LinkTo>** for all web properties. Review those standards before starting a new project so your implementation aligns with the organizational approach.

## What is mangrove?

Mangrove is UNDRR's component library that provides:

- Pre-built, tested components ready for production
- Consistent design across all UNDRR projects
- WCAG 2.1 AA accessibility compliance
- Works with any tech stack (React, Drupal, static HTML)
- Copy-paste ready code snippets
- Live examples and documentation for every component

## Quick start

The fastest way to get started:

```html
<!DOCTYPE html>
<html>
  <head>
    <!-- Base styling (choose one theme) -->
    <link
      rel="stylesheet"
      href="https://assets.undrr.org/static/mangrove/1.3.3/css/style.css"
    />
    <!-- Alternative themes:
    <link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style-mcr.css" />
    <link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style-irp.css" />
    <link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style-preventionweb.css" />
    -->
  </head>
  <body>
    <!-- Use a component -->
    <button class="mg-cta-button mg-cta-button--primary">
      <span class="mg-cta-button__text">Get Started</span>
      <span class="mg-cta-button__icon" aria-hidden="true">→</span>
    </button>

    <!-- Widget loader script -->
    <script
      type="text/javascript"
      src="https://publish.preventionweb.net/widget.js?rand=3d797b"
    ></script>

    <!-- Widget initialization -->
    <script>
      new PW_Widget.initialize({
        contenttype: 'landingpage',
        pageid: '83835',
        includemetatags: false,
        includecss: true,
        suffixID: 'footer',
        activedomain: 'www.undrr.org',
      });
    </script>

    <!-- UNDRR Analytics - GA4 bootstrap and enhancements -->
    <script
      src="https://assets.undrr.org/static/analytics/v1.0.0/google_analytics_enhancements.js"
      defer
    ></script>

    <!-- UNDRR Critical Messaging - emergency broadcasts -->
    <script
      src="https://messaging.undrr.org/src/undrr-messaging.js"
      defer
    ></script>
  </body>
</html>
```

## Integration approaches

Choose the approach that best fits your project:

### Approach 1: Vanilla HTML/CSS (simplest)

**Best for**: Static sites, existing projects, quick prototypes

- Include CSS files via CDN
- Browse the component catalog and select what you need
- Check the component's documentation page
- Copy HTML structures from component docs
- Add the referenced CSS and JS files to your project as specified in the documentation
- No JavaScript required for most components

<LinkTo kind="getting-started-vanilla-html-and-css" story="docs">
  → See Vanilla HTML/CSS Guide
</LinkTo>

### Approach 2: Sass integration

**Best for**: Custom themes, design system extensions

- Import Sass files into your build process
- Customize variables and mixins
- Build your own theme variants

<LinkTo kind="getting-started-sass-integration" story="docs">
  → See Sass Integration Guide
</LinkTo>

### Approach 3: React components

**Best for**: React applications, dynamic interfaces

- Import React components directly
- Full TypeScript support
- Props-based customization
- Built-in state management

### Installation via yarn or npm

Install the library via npm or yarn:

```bash
# NPM
npm install @undrr/undrr-mangrove

# Yarn
yarn add @undrr/undrr-mangrove
```

### Checking and updating to the latest version

To check the current latest version available on npm:

```bash
npm view @undrr/undrr-mangrove version
# or
npm info @undrr/undrr-mangrove version
```

To install the latest version:

```bash
npm install @undrr/undrr-mangrove@latest
```

To update all dependencies to their latest semver-compatible versions:

```bash
npm outdated
npm update
```

### Approach 4: Server-rendered hydration

**Best for**: Drupal, Astro, or any server-rendered framework that needs React interactivity

- Server renders HTML containers with `data-mg-*` attributes
- The `createHydrator` runtime finds containers, extracts props, and mounts React components
- Each component ships a `fromElement` function for prop extraction — no hand-written boilerplate per wrapper

[→ See Hydration guide on GitHub](https://github.com/unisdr/undrr-mangrove/blob/main/docs/HYDRATION.md)

## Architecture overview

Mangrove follows a loose [atomic design](https://atomicdesign.bradfrost.com/) methodology:

```
Atoms (building blocks)
├── Typography (headings, paragraphs, links)
├── Colors (brand palette, semantic colors)
├── Icons (consistent iconography)
└── Layout (grid, spacing, containers)

Molecules (simple combinations)
├── Header navigation
├── Footer sections
└── Content blocks

Components (complex functional units)
├── Buttons (CTAs, chips, share buttons)
├── Forms (inputs, selects, checkboxes)
├── Cards (content cards, stats cards)
├── Navigation (breadcrumbs, menus)
└── Charts (bar charts, scatter plots)
```

## Key principles

### Framework agnostic

Mangrove works with any technology stack: React, Vue, Angular, plain HTML, WordPress, Drupal, etc.

- Minimal dependencies, so it won't conflict with your existing setup
- Repeatable HTML structures keep the look and feel consistent across implementations

### CSS namespace

All styles use the `mg-` prefix to prevent conflicts with your existing codebase:

```css
.mg-button {
  /* Mangrove button */
}
.mg-card {
  /* Mangrove card */
}
.mg-form__input {
  /* Mangrove form input */
}
```

### Semantic HTML

Components use proper HTML elements for accessibility and maintain a consistent structure:

1. Each component uses the `mg-` prefix in its class names
2. Semantic HTML elements are used where appropriate
3. ARIA attributes are included for accessibility

```html
<button class="mg-cta-button">Button</button>
<nav class="mg-breadcrumb">Navigation</nav>
<main class="mg-content">Main content</main>
```

Example component structure:

```html
<div class="mg-component">
  <div class="mg-component__element">
    <!-- Component content -->
  </div>
</div>
```

### Progressive enhancement

Components work without JavaScript, enhanced with JS where beneficial.

## Common use cases

### Marketing pages

```html
<!-- Hero section with CTA -->
<section class="mg-hero">
  <div class="mg-container">
    <h1 class="mg-heading mg-heading--1">Building Resilient Communities</h1>
    <p class="mg-paragraph mg-paragraph--large">
      Disaster risk reduction starts here.
    </p>
    <button class="mg-cta-button mg-cta-button--primary">
      <span class="mg-cta-button__text">Learn More</span>
    </button>
  </div>
</section>
```

### Data dashboards

```jsx
import { BarChart, StatsCard } from '@undrr/undrr-mangrove';

function Dashboard() {
  const chartData = [
    { label: '2020', value: 389 },
    { label: '2021', value: 412 },
    { label: '2022', value: 475 },
  ];

  return (
    <div className="mg-grid">
      <StatsCard
        title="Disaster impact"
        stats={[
          { label: 'Events', value: '475', bottomLabel: 'in 2022' },
          { label: 'Economic losses', value: '$223B', bottomLabel: 'estimated' },
        ]}
      />
      <BarChart data={chartData} />
    </div>
  );
}
```

### Forms and applications

<form class="mg-form">

## Development setup

The steps below cover the basics for getting your development environment running. For more in-depth details on our development process, including our branching strategy, testing with Chromatic, and using Makefile commands, check out the [project `README.md` on GitHub](https://github.com/unisdr/undrr-mangrove?tab=readme-ov-file#development).

1. **Clone the repository**

```bash
git clone https://github.com/unisdr/undrr-mangrove.git
cd undrr-mangrove
```

2. **Use Docker for development (recommended)**

```bash
# Start the application, runs Storybook locally on port 6006
yarn docker-run
```

Alternatively, if you prefer not to use Docker:

```bash
# Install dependencies
yarn install

# Run Storybook locally
yarn run storybook
```

3. **Building the project**

```bash
# Using Docker
yarn docker-build

# Without Docker
yarn run build
```

4. **Testing**

```bash
# Run tests
yarn test

# Run with coverage report
yarn test:coverage
```

5. **Linting**

```bash
# Run linting
yarn run lint
```
