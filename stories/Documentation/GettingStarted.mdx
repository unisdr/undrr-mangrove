## What is Mangrove?

Mangrove is UNDRR's component library for consistent, accessible web development. Read **<LinkTo kind="getting-started-about-mangrove" story="docs">About Mangrove</LinkTo>** to understand what's shared across UNDRR web properties and where your project fits.

Mangrove provides:

- Pre-built, tested components ready for production
- Shared content structures across UNDRR properties
- WCAG 2.2 AA accessibility compliance
- Works with any tech stack (React, Drupal, static HTML)
- Copy-paste ready code snippets
- Live examples and documentation for every component

## Integration approaches

Choose the approach that fits your project. Each guide has its own quick start with working code examples.

### Vanilla HTML/CSS (simplest)

Include CSS via CDN and copy HTML structures from the component docs. No build step, no JavaScript required for most components.

```html
<link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.6.0/css/style.css" />
```

- [See on CodePen](https://codepen.io/khawkins98/pen/MYwbKwe) — a working page with footer widget, analytics, and messaging
- <LinkTo kind="getting-started-integration-vanilla-html-and-css" story="docs">Vanilla HTML/CSS guide</LinkTo>

### Sass integration

Import SCSS source files into your build process for custom themes and variable overrides.

- <LinkTo kind="getting-started-integration-sass-integration" story="docs">Sass integration guide</LinkTo>

### React components

Install via npm and import React components with props-based customization. The npm package ships pre-compiled components with React externalized.

```bash
yarn add @undrr/undrr-mangrove
```

- <LinkTo kind="getting-started-integration-react-integration" story="docs">React integration guide</LinkTo>
- A full yarn/npm-installed demo project is in progress ([#849](https://github.com/unisdr/undrr-mangrove/issues/849))

### Hydration

For Drupal, Astro, or any server-rendered framework that needs React interactivity. Server renders HTML containers with `data-mg-*` attributes; the `createHydrator` runtime mounts React components into them.

- <LinkTo kind="getting-started-integration-hydration-guide" story="docs">Hydration guide</LinkTo>

### CDN and versioning

All assets are served from versioned CDN endpoints. See the <LinkTo kind="getting-started-integration-cdn-reference" story="docs">CDN reference</LinkTo> for URLs and versioning details.

## Key principles

- All CSS classes use the `mg-` prefix (e.g., `mg-card`, `mg-form__input`) to avoid conflicts with your styles
- Components use semantic HTML with ARIA attributes for accessibility
- Most components work without JavaScript (progressive enhancement)
- Supports modern evergreen browsers -- see <LinkTo kind="getting-started-browser-support" story="docs">browser support</LinkTo>
- Supports right-to-left (RTL) layouts -- components adapt automatically when `dir="rtl"` is set on the document

For the full architectural picture, see the <LinkTo kind="contributing-architecture" story="docs">architecture guide</LinkTo>.
