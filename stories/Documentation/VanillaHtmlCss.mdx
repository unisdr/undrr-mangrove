## Quick start

### 1. Include CSS files

Add these `<link>` tags to your HTML `<head>`:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My UNDRR Site</title>

    <!-- UNDRR Theme (Choose one) -->
    <link
      rel="stylesheet"
      href="https://assets.undrr.org/static/mangrove/1.3.3/css/style.css"
    />

    <!-- Alternative: Prevention Web Theme -->
    <!-- <link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style-preventionweb.css"> -->
  </head>
  <body>
    <!-- Your content here -->

    <!-- Widget loader script -->
    <script
      type="text/javascript"
      src="https://publish.preventionweb.net/widget.js?rand=3d797b"
    ></script>

    <!-- Widget initialization -->
    <script>
      new PW_Widget.initialize({
        contenttype: 'landingpage',
        pageid: '83835',
        includemetatags: false,
        includecss: false,
        suffixID: 'footer',
        activedomain: 'www.undrr.org',
      });
    </script>
  </body>
</html>
```

### 2. Copy HTML structures

Browse the component library, click the "HTML" tab, and copy the markup:

```html
<!-- Example: Primary Button -->
<a class="mg-button mg-button-primary" role="button" href="#">
  Download Report
</a>
```

## Best practices

### 1. Semantic HTML first

Always use the correct HTML element:

```html
<!-- Good -->
<button class="mg-cta-button">Submit</button>
<nav class="mg-breadcrumb">...</nav>

<!-- Avoid -->
<div class="mg-cta-button">Submit</div>
<div class="mg-breadcrumb">...</div>
```

### 2. Accessibility attributes

Include proper ARIA labels and attributes:

```html
<button class="mg-cta-button" aria-label="Download PDF report">
  <span class="mg-cta-button__text">Download</span>
  <span class="mg-cta-button__icon" aria-hidden="true">↓</span>
</button>
```

### 3. Responsive design

Use responsive utility classes:

```html
<div class="mg-grid mg-grid--1-col mg-grid--md-2-cols mg-grid--lg-3-cols">
  <!-- Content automatically adapts to screen size -->
</div>
```

## JavaScript

Most Mangrove components work without JavaScript. For enhanced interactivity, you have two options.

### Vanilla JS modules

Some components (like Tabs) have lightweight JavaScript modules:

```html
<script type="module">
  import { mgTabs } from 'https://assets.undrr.org/static/mangrove/1.3.3/js/tabs.js';

  document.addEventListener('DOMContentLoaded', () => {
    mgTabs();
  });
</script>
```

### Precompiled React components

For complex interactive components (MegaMenu, charts, maps), use precompiled React bundles without a build process.

**Available components:**

| Component | CDN Path | Use case |
|-----------|----------|----------|
| ShareButtons | `/components/ShareButtons.js` | Social sharing |
| MegaMenu | `/components/MegaMenu.js` | Full navigation with dropdowns |
| ScrollContainer | `/components/ScrollContainer.js` | Horizontal scrolling |
| BarChart | `/components/BarChart.js` | Data visualization |
| MapComponent | `/components/MapComponent.js` | Interactive maps |
| QuoteHighlight | `/components/QuoteHighlight.js` | Styled pull quotes |
| Fetcher | `/components/Fetcher.js` | Data fetching wrapper |
| SyndicationSearchWidget | `/components/SyndicationSearchWidget.js` | PreventionWeb search |
| IconCard | `/components/IconCard.js` | Icon-based info cards |
| Gallery | `/components/Gallery.js` | Image gallery with thumbnails |
| StatsCard | `/components/StatsCard.js` | Statistics display |
| Pager | `/components/Pager.js` | Pagination controls |

React 19 dropped UMD builds, so we use [import maps](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/script/type/importmap) with [esm.sh](https://esm.sh/) to provide React dependencies:

```html
<script type="importmap">
  {
    "imports": {
      "react": "https://esm.sh/react@19.2.4",
      "react/jsx-runtime": "https://esm.sh/react@19.2.4/jsx-runtime",
      "react-dom": "https://esm.sh/react-dom@19.2.4",
      "react-dom/": "https://esm.sh/react-dom@19.2.4/"
    }
  </script>
  <!--
    Note: react/jsx-runtime is only needed if components use the automatic JSX
    runtime. Many components don't require it. Import maps only trigger network
    requests when code actually imports from that path, so unused entries are harmless.
  -->
</head>
<body>
  <!-- Mount point -->
  <nav data-mg-mega-menu></nav>

  <script type="module">
    import React from 'react';
    import { createRoot } from 'react-dom/client';

    // Load component from CDN
    const MegaMenuModule = await import(
      'https://assets.undrr.org/static/mangrove/1.3.3/components/MegaMenu.js'
    );

    // Unwrap ESM/CJS interop - bundle may be double-wrapped as { default: { default: Component } }
    let MegaMenu = MegaMenuModule?.default ?? MegaMenuModule;
    if (typeof MegaMenu !== 'function' && MegaMenu?.default) {
      MegaMenu = MegaMenu.default;
    }

    // Render the component
    const container = document.querySelector('[data-mg-mega-menu]');
    if (container) {
      const root = createRoot(container);
      root.render(
        React.createElement(MegaMenu, {
          delay: 2000,
          sections: [
            {
              title: 'About',
              bannerHeading: 'About Us',
              bannerDescription: 'Learn more about our organization.',
              items: [
                { title: 'Our Mission', url: '/about/mission' },
                { title: 'Our Team', url: '/about/team' }
              ]
            },
            {
              title: 'Search',
              bannerButton: { url: '/search' }
            }
          ]
        })
      );
    }
  </script>
</body>
```

**Tip:** For debugging, use dev builds by adding `?dev` to esm.sh URLs (e.g., `https://esm.sh/react@19.2.4?dev`).

### Hydration helper (recommended)

Components that support hydration (ShareButtons, ScrollContainer, QuoteHighlight, Gallery, IconCard, StatsCard, MegaMenu, SyndicationSearchWidget) export a `fromElement` function that extracts props from data attributes. Combined with the `createHydrator` helper, this eliminates all manual `createRoot` boilerplate:

```html
<script type="module">
  import createHydrator from 'https://assets.undrr.org/static/mangrove/1.3.3/components/hydrate.js';
  import ShareButtons, { fromElement } from 'https://assets.undrr.org/static/mangrove/1.3.3/components/ShareButtons.js';

  createHydrator({
    selector: '[data-mg-share-buttons]',
    component: ShareButtons,
    fromElement,
  });
</script>

<!-- Container with props as data attributes -->
<section data-mg-share-buttons
  data-main-label="Share this"
  data-on-copy-label="Link copied"
  data-sharing-subject="Sharing Link">
</section>
```

This pattern works for any component with hydration support. See the [Hydration guide](https://github.com/unisdr/undrr-mangrove/blob/main/docs/HYDRATION.md) for the full API, supported components, and how to add hydration to new components.

### Manual mounting (for components without hydration support)

Components that don't yet export `fromElement` (BarChart, MapComponent, etc.) can be mounted manually with `createRoot`:

```html
<nav data-mg-mega-menu></nav>

<script type="module">
  import React from 'react';
  import { createRoot } from 'react-dom/client';
  const MegaMenuModule = await import(
    'https://assets.undrr.org/static/mangrove/1.3.3/components/MegaMenu.js'
  );
  let MegaMenu = MegaMenuModule?.default ?? MegaMenuModule;
  if (typeof MegaMenu !== 'function' && MegaMenu?.default) {
    MegaMenu = MegaMenu.default;
  }
  const container = document.querySelector('[data-mg-mega-menu]');
  if (container) {
    createRoot(container).render(React.createElement(MegaMenu, { /* props */ }));
  }
</script>
```

For most use cases, the vanilla HTML markup (shown in component docs) is simpler. Use precompiled components only when you need:
- Complex interactive behavior (MegaMenu dropdowns, chart interactions)
- Dynamic data loading
- State management

[See complete working example on CodePen](https://codepen.io/khawkins98/pen/MYwbKwe) — includes React MegaMenu, header, hero, tabs, footer, analytics, and messaging.

## Complete HTML page

A self-contained page you can save as an `.html` file and open in a browser. It loads the UNDRR theme, sets up an import map for React, and renders a ShareButtons component using the hydration helper.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Mangrove component demo</title>

    <!-- Mangrove CSS theme -->
    <link
      rel="stylesheet"
      href="https://assets.undrr.org/static/mangrove/1.3.3/css/style.css"
    />

    <!-- Import map: provides React to Mangrove ES module bundles -->
    <script type="importmap">
      {
        "imports": {
          "react": "https://esm.sh/react@19.2.4",
          "react/jsx-runtime": "https://esm.sh/react@19.2.4/jsx-runtime",
          "react-dom": "https://esm.sh/react-dom@19.2.4",
          "react-dom/": "https://esm.sh/react-dom@19.2.4/"
        }
      }
    </script>
  </head>
  <body>
    <main class="mg-container">
      <h1>Share this page</h1>

      <!-- Mount point: the hydrator finds this element by its data attribute -->
      <section
        data-mg-share-buttons
        data-main-label="Share this"
        data-on-copy-label="Link copied"
        data-sharing-subject="Mangrove demo page"
      ></section>
    </main>

    <!-- Load the hydration runtime and component, then mount -->
    <script type="module">
      import createHydrator from 'https://assets.undrr.org/static/mangrove/1.3.3/components/hydrate.js';
      import ShareButtons, { fromElement } from 'https://assets.undrr.org/static/mangrove/1.3.3/components/ShareButtons.js';

      createHydrator({
        selector: '[data-mg-share-buttons]',
        component: ShareButtons,
        fromElement,
      });
    </script>
  </body>
</html>
```

To use a different component, swap the import path and selector. Components with hydration support (ShareButtons, MegaMenu, ScrollContainer, QuoteHighlight, Gallery, IconCard, StatsCard, SyndicationSearchWidget) all follow this same pattern. Components without hydration support (BarChart, MapComponent, Fetcher, Pager) require manual mounting with `createRoot` as shown in the [manual mounting](#manual-mounting-for-components-without-hydration-support) section above.

## UNDRR integration

All UNDRR pages should include analytics and messaging scripts.

### Analytics

Track user behavior with GA4 enhancements:

```html
<script
  src="https://assets.undrr.org/static/analytics/v1.0.0/google_analytics_enhancements.js"
  defer
></script>
```

### Critical messaging

Receive emergency broadcasts across all UNDRR properties:

```html
<script src="https://messaging.undrr.org/src/undrr-messaging.js" defer></script>
```

Optionally add a container for message placement:

```html
<div class="mg-critical-messaging"></div>
```

See [Critical messaging](/?path=/docs/getting-started-critical-messaging--docs) for full documentation.

## Troubleshooting

### Import map browser support

Import maps are supported in Chrome 89+, Safari 16.4+, and Firefox 108+. For older browsers, add the [es-module-shims](https://github.com/guybedford/es-module-shims) polyfill before your import map:

```html
<script async src="https://ga.jspm.io/npm:es-module-shims@1.10.1/dist/es-module-shims.js"></script>
<script type="importmap">
  {
    "imports": {
      "react": "https://esm.sh/react@19.2.4",
      "react/jsx-runtime": "https://esm.sh/react@19.2.4/jsx-runtime",
      "react-dom": "https://esm.sh/react-dom@19.2.4",
      "react-dom/": "https://esm.sh/react-dom@19.2.4/"
    }
  }
</script>
```

### CORS errors when self-hosting

If you host Mangrove JS files on your own server, the browser may block cross-origin imports. Ensure your server sends the correct headers:

```
Access-Control-Allow-Origin: *
Content-Type: application/javascript
```

When serving from `assets.undrr.org`, CORS headers are already configured.

### Component renders as blank

If a `data-mg-*` container stays empty after the page loads, check these items in order:

1. **Import map present?** The `<script type="importmap">` block must appear before any `<script type="module">` tags. Browsers ignore import maps added after the first module script runs.
2. **React version mismatch?** The `react` and `react-dom` versions in the import map must match. Mixing versions causes silent failures.
3. **Correct selector?** The `selector` passed to `createHydrator` must match the `data-mg-*` attribute on the container element. Open the browser console and look for errors from the hydration runtime.
4. **Network failures?** Open DevTools > Network tab and check that all `.js` files loaded with status 200. A 404 on the component or `hydrate.js` will prevent rendering.
5. **Multiple import maps?** Browsers only support one `<script type="importmap">` per page. If you have multiple, only the first one applies.
6. **Console errors?** `createHydrator` logs errors to `console.error` and restores the original container HTML on failure. Check the browser console for details.
