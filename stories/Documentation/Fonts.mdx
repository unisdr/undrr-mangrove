# Fonts

## Roboto

Roboto is the primary font family used throughout Mangrove for body text and form content.

```scss
$mg-font-family: "Roboto", sans-serif;
```

**Usage:**
- Body text and paragraphs
- Form inputs and form labels
- Default for all text content

### Roboto Condensed

Roboto Condensed is reserved for **compact UI labels and decorative chrome**, not for narrative headings. Long-form headings (h1–h6 used in articles, story content, page bodies) inherit the body face — Roboto Condensed at h1/h2 size feels too tight for sustained reading.

Two SCSS tokens cover the condensed-font use cases:

```scss
// Semantic alias — use this for component-level title/label classes that act
// as visual headings within a card, hero, tab, or other UI chrome context.
$mg-font-family-headings: $mg-font-family-condensed !default;

// Raw token — use when compact width is the goal and the element is not
// a heading (e.g. dense navigation that needs to fit more characters).
$mg-font-family-condensed: "Roboto Condensed", sans-serif;
```

**When to use which:**

| Token | Use for |
|---|---|
| (default body font) | h1–h6 in narrative content. They inherit Roboto from `body`. Do not declare a `font-family` on h1–h6 in component SCSS. |
| `$mg-font-family-headings` | Component title/label classes: `.mg-hero__title`, `.mg-hero__label`, `.mg-card__title`, `.mg-card__label`, `.mg-stats-card-item__value`, `.mg-stats-card-item__label`, `.mg-stats-card-item__bottom-label`, `.mg-gallery__title`, `.mg-tabs__link`, `.mg-on-this-page-nav__link`, `.mg-on-this-page-nav__cta`, `.mg-mega-topbar__item-link`, `.mg-mega-content__banner header`, `.mg-button` (CTA buttons), `.mg-footer--about-footer--links`, `.mg-tag`, and any new component class that functions as a chip, label, or compact title within UI chrome. |
| `$mg-font-family-condensed` | Dense UI chrome where compact width is the explicit design goal, not heading semantics. Currently: breadcrumbs. |

If a class is a visual title or label inside a component (not part of narrative body content), prefer `$mg-font-family-headings`. The `!default` flag on the token lets a consumer theme remap the family without forking the SCSS.

## Font sources

All fonts are served from the UNDRR CDN for reliability and performance:

| Font | CDN Path |
|------|----------|
| Roboto | `https://assets.undrr.org/static/fonts/roboto/v1.0.0/` |
| Roboto Condensed | `https://assets.undrr.org/static/fonts/roboto-condensed/v1.0.0/` |

## Performance

All font declarations use `font-display: swap` to prevent invisible text during load. Text renders immediately with a fallback font, then swaps to the web font once loaded.

## Language-specific fonts

Mangrove supports language-specific font overrides for scripts that require specialized typography. These fonts are applied automatically based on the `lang` attribute.

### How it works

When content is marked with a language attribute (e.g., `lang="ar"`), CSS `:lang()` selectors automatically apply the appropriate fonts:

```html
<div lang="ar">
  <!-- Arabic fonts applied automatically -->
</div>
```

### Supported languages

| Language | Selector | Font override |
|----------|----------|---------------|
| Arabic | `:lang(ar)` | Noto Kufi Arabic (headings), Dubai (body) |

## Arabic fonts

Arabic content uses dedicated fonts optimized for Arabic script, replacing Roboto which lacks proper Arabic character support.

### Font usage

| Purpose | Font | Weight | Variable |
|---------|------|--------|----------|
| Headings (h1-h6) | Noto Kufi Arabic | 700 only | `$mg-font-family-arabic-headings` |
| Body text | Dubai | 400 | `$mg-font-family-arabic-body` |

> **Note**: Noto Kufi Arabic is only available in 700 (Bold) weight, which is appropriate for headings. Dubai is used for body text at regular weight.

### Implementation

Add the `lang="ar"` attribute to enable Arabic fonts:

```html
<div lang="ar" dir="rtl">
  <h1>عنوان رئيسي</h1>
  <p>هذا نص تجريبي باللغة العربية.</p>
</div>
```

To see Arabic fonts in action, [view the Hero component with Arabic locale](?path=/story/components-hero-hero--default&globals=locale:arabic;addonRtl:rtl).

### Arabic font sources

> **Note**: Fonts are temporarily served from the testing CDN during development. Before production release, update paths to use `/static/fonts/` instead of `/testing/static/fonts/`.

| Font | Weight | CDN Path |
|------|--------|----------|
| Noto Kufi Arabic | 700 (Bold) | `https://assets.undrr.org/testing/static/fonts/noto-kufi-arabic/v1.0.0/Bold/` |
| Dubai | 400 (Regular) | `https://assets.undrr.org/testing/static/fonts/dubai/v1.0.0/Regular/` |

**Font formats**: WOFF2 and WOFF formats are served for optimal browser compatibility and compression.

### RTL support

Arabic fonts and RTL layout are handled separately:

- **`lang="ar"`** - Triggers Arabic font styles via `:lang(ar)` selector
- **`dir="rtl"`** - Triggers RTL layout adjustments (padding, margins, transforms)

For full Arabic support, use both attributes:

```html
<div lang="ar" dir="rtl">
  <h1>عنوان رئيسي</h1>
  <p>نص من اليمين إلى اليسار</p>
</div>
```

## Component development

When creating components that explicitly set `font-family`, add language-specific overrides to ensure proper rendering.

### Pattern for Arabic fonts

```scss
// Component styles
.my-component {
  font-family: $mg-font-family;
}

// Arabic font override (add at end of component file)
:lang(ar) {
  .my-component {
    font-family: $mg-font-family-arabic-body;
  }
}
```
