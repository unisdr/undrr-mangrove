Use Mangrove React components in your application for props-driven UI with built-in accessibility and UNDRR brand compliance.

This guide assumes familiarity with **<LinkTo kind="getting-started-what-mangrove-is" story="docs">What Mangrove is</LinkTo>**. Review that first if you're working on a project that may need custom approaches.

## Installation

```bash
yarn add @undrr/undrr-mangrove
```

Mangrove requires React 18 or later as a peer dependency. If you do not already have React installed:

```bash
yarn add react react-dom
```

## Importing components

Import components by name from the package root:

```jsx
import { StatsCard, Pager, ShareButtons } from '@undrr/undrr-mangrove';
```

You can also import the hydration runtime if you need to mount components into server-rendered HTML containers:

```jsx
import { createHydrator } from '@undrr/undrr-mangrove';
```

## Including styles

Mangrove does not ship pre-built CSS in the npm package at a conventional `dist/css/` path. There are two ways to include styles:

**Option A: CDN (recommended for most consumers)**

Load the compiled CSS from UNDRR's CDN. Pick the theme that matches your site:

```html
<!-- UNDRR default theme -->
<link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style.css" />

<!-- Or a site-specific theme -->
<link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style-preventionweb.css" />
```

Available theme files: `style.css`, `style-preventionweb.css`, `style-mcr.css`, `style-irp.css`.

See the <LinkTo kind="getting-started-cdn-reference" story="docs">CDN reference</LinkTo> for the full list of available assets and versioned URLs.

**Option B: Sass integration**

If your build pipeline supports Sass, you can import the source SCSS files directly and customize variables. See the <LinkTo kind="getting-started-sass-integration" story="docs">Sass integration guide</LinkTo> for setup instructions.

## Usage examples

### StatsCard

```jsx
import { StatsCard } from '@undrr/undrr-mangrove';

function DashboardStats() {
  return (
    <StatsCard
      title="Global impact"
      variant="default"
      stats={[
        {
          icon: 'mg-icon mg-icon-globe',
          label: 'Target A',
          value: '195',
          bottomLabel: 'UN member states',
          summaryText: 'Working together on disaster risk reduction',
        },
        {
          value: '45%',
          label: 'Target B',
          bottomLabel: 'Reduction in mortality',
        },
      ]}
    />
  );
}
```

Props: `title` (string), `stats` (array of objects with `icon`, `label`, `value`, `bottomLabel`, `summaryText`, `link`), `variant` (`"default"` | `"compact"` | `"highlighted"` | `"negative"`), `className`.

### Pager

```jsx
import { useState } from 'react';
import { Pager } from '@undrr/undrr-mangrove';

function PaginatedList() {
  const [page, setPage] = useState(1);

  return (
    <div>
      {/* Your list content here */}
      <Pager
        page={page}
        totalPages={10}
        onPageChange={setPage}
      />
    </div>
  );
}
```

Props: `page` (number, required), `totalPages` (number), `onPageChange` (function, required), `isLoading` (boolean), `layout` (`"centered"` | `"bar"`), `range` (object with `start` and `end`), `showJumpTo` (boolean), `emptyState` (string).

### ShareButtons

```jsx
import { ShareButtons } from '@undrr/undrr-mangrove';

function ArticlePage() {
  return (
    <ShareButtons
      labels={{
        mainLabel: 'Share this page',
        onCopy: 'Copied',
      }}
      SharingSubject="Disaster risk reduction report"
      SharingTextBody="Check out this report: "
    />
  );
}
```

Props: `labels` (object with `mainLabel` and `onCopy` strings), `SharingSubject` (string), `SharingTextBody` (string).

### QuoteHighlight

```jsx
import { QuoteHighlight } from '@undrr/undrr-mangrove';

function QuoteSection() {
  return (
    <QuoteHighlight
      quote="Disaster risk reduction is everyone's business."
      attribution="Mami Mizutori"
      attributionTitle="Special Representative of the Secretary-General"
      variant="line"
      backgroundColor="light"
    />
  );
}
```

Props: `quote` (string, required), `attribution` (string), `attributionTitle` (string), `imageSrc` (string), `imageAlt` (string), `variant` (`"line"` | `"image"`), `backgroundColor` (`"light"` | `"dark"` | `"bright"`), `alignment` (`"full"` | `"left"` | `"right"`).

### Snackbar

```jsx
import { useState } from 'react';
import { Snackbar } from '@undrr/undrr-mangrove';

function FormWithFeedback() {
  const [open, setOpen] = useState(false);

  return (
    <div>
      <button onClick={() => setOpen(true)}>Submit</button>
      <Snackbar
        severity="success"
        opened={open}
        message="Your changes have been saved."
        onClose={() => setOpen(false)}
        openedMiliseconds={5000}
      />
    </div>
  );
}
```

Props: `severity` (`"error"` | `"warning"` | `"info"` | `"success"`), `opened` (boolean), `message` (string), `onClose` (function), `openedMiliseconds` (number).

## Server-rendered hydration

If you are rendering HTML on the server (Drupal, Astro, or any other framework) and need React to take over specific DOM containers, Mangrove provides `createHydrator`. This scans the page for elements matching a CSS selector, extracts props from `data-` attributes, and mounts the component.

The following example shows the **Drupal wrapper pattern**, where a small script imports component bundles via import map aliases (`@mangrove/*`) that resolve to CDN URLs at runtime. The import map is injected by `mangrove-components.js` in the `undrr_common` theme. In a typical React app, you would import from `@undrr/undrr-mangrove` instead and pass props directly (see the [usage examples](#usage-examples) above).

```jsx
// Drupal wrapper script â€” uses @mangrove/* import map aliases (resolved by mangrove-components.js)
import createHydrator from "@mangrove/hydrate";
import ShareButtons, { fromElement } from "@mangrove/ShareButtons";

const hydrator = createHydrator({
  selector: '[data-mg-share-buttons]',
  component: ShareButtons,
  fromElement,
});

// Re-scan when Drupal adds new DOM (AJAX, BigPipe, modals, Views infinite scroll)
Drupal.behaviors.mangroveShareButtons = {
  attach(context) {
    hydrator.update(context);
  },
};
```

The `fromElement` function exported by ShareButtons extracts props from the container's data attributes (`data-main-label`, `data-on-copy-label`, `data-sharing-subject`, `data-sharing-body`). Each component with hydration support ships its own `fromElement` â€” you do not need to write one by hand.

See the [hydration guide](https://github.com/unisdr/undrr-mangrove/blob/main/docs/HYDRATION.md) for the full API, including `update()`, `unmountAll()`, and error handling options.

## TypeScript

TypeScript is supported in the Mangrove development environment, but the published npm package does not yet ship type declarations (`.d.ts` files). If you use TypeScript in your consuming project, you can declare the module yourself:

```ts
// mangrove.d.ts
declare module '@undrr/undrr-mangrove' {
  import { ComponentType } from 'react';

  export const StatsCard: ComponentType<{
    title?: string;
    stats?: Array<{
      icon?: string;
      label?: string;
      value: string | number;
      bottomLabel?: string;
      summaryText?: string;
      link?: string;
    }>;
    variant?: 'default' | 'compact' | 'highlighted' | 'negative';
    className?: string;
  }>;

  export const Pager: ComponentType<{
    page: number;
    totalPages?: number;
    onPageChange: (page: number) => void;
    isLoading?: boolean;
    layout?: 'centered' | 'bar';
    ariaLabel?: string;
    range?: { start: number; end: number };
    rangeLabel?: string;
    showJumpTo?: boolean;
    jumpToLabel?: string;
    jumpToAction?: string;
    emptyState?: string;
    emptyAction?: { label: string; onClick: () => void };
  }>;

  // Add declarations for other components as needed
}
```

Refer to each component's PropTypes (visible in Storybook's controls panel) for the full list of accepted props.

## Testing

Components from Mangrove can be tested with React Testing Library like any other React component:

```jsx
import { render, screen } from '@testing-library/react';
import { Pager } from '@undrr/undrr-mangrove';

test('renders pagination with correct page', () => {
  const handlePageChange = jest.fn();

  render(
    <Pager page={3} totalPages={10} onPageChange={handlePageChange} />
  );

  expect(screen.getByRole('navigation', { name: /pagination/i })).toBeInTheDocument();
});
```
