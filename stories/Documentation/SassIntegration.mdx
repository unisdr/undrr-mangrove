## Installation

```bash
yarn add @undrr/undrr-mangrove

# or
npm install @undrr/undrr-mangrove
```

The SCSS source files are in `stories/assets/scss/` within the installed package.

## SCSS file structure

Mangrove's styles are organized as a set of SCSS partials imported by a theme entry point. The default theme is `style.scss`:

```
stories/assets/scss/
  style.scss                   # Default (UNDRR) theme entry point
  style-preventionweb.scss     # PreventionWeb theme
  style-irp.scss               # IRP theme
  style-mcr.scss               # MCR2030 theme
  style-delta.scss             # DELTA Resilience theme
  style-gutenberg.scss         # Legacy Gutenberg editor subset
  _variables.scss              # Design tokens and component variables
  _variables-preventionweb.scss # PW variable overrides
  _variables-irp.scss          # IRP variable overrides
  _variables-mcr.scss          # MCR variable overrides
  _variables-delta.scss        # DELTA Resilience variable overrides
  _fonts.scss                  # Web font imports
  _breakpoints.scss            # devicebreak() mixin
  _mixins.scss                 # Vendor-prefix and utility mixins
  _utility.scss                # CSS utility classes
  _foundational.scss           # Base typography and element styles
  _components.scss             # Rollup of all component SCSS imports
```

Each theme entry point follows the same import order:

```scss
@import "./variables";            // Base tokens
@import "./variables-{theme}";    // Theme-specific overrides (omitted in default)
@import "./fonts";
@import "./breakpoints";
@import "./mixins";
@import "./utility";
@import "./foundational";
@import "./components";
```

`_variables.scss` defines tokens with the `!default` flag, so a theme file imported after it can reassign those variables before they reach the component stylesheets.

## Mixins reference

### `devicebreak` (from `_breakpoints.scss`)

Responsive media query mixin. This is the primary breakpoint utility used throughout Mangrove.

```scss
@mixin devicebreak($point) { ... }
```

**Breakpoint values:**

| Name | Media query | Approximate size |
|------|------------|-----------------|
| `small` | `width <= 47.9375em` | Up to ~767px |
| `mediumonlytab` | `width <= 89.9375em` | Up to ~1439px |
| `medium` | `width >= 48em` | 768px and up |
| `large` | `width >= 64em` | 1024px and up |
| `xlarge` | `width >= 90em` | 1440px and up |

**Usage:**

```scss
.my-component {
  padding: $mg-spacing-100;

  @include devicebreak(medium) {
    padding: $mg-spacing-200;
  }

  @include devicebreak(xlarge) {
    padding: $mg-spacing-300;
  }
}
```

### `mg-typography` (from `_variables.scss`)

Sets font size with associated padding for heading-like elements.

```scss
@mixin mg-typography($size) { ... }
```

**Size values:** `"extra-small"`, `"small"`, `"medium"`, `"large"`, `"xlarge"`, `"huge"`, `"xhuge"`

**Usage:**

```scss
.my-heading {
  @include mg-typography("large");
  // Outputs: font-size: 3.2rem; padding-top: 2.4rem; padding-bottom: 1.5rem;
}
```

### Vendor-prefix mixins (from `_mixins.scss`)

These mixins add vendor prefixes for CSS properties. Note that if your build pipeline already includes Autoprefixer, you may not need these.

| Mixin | Signature | Purpose |
|-------|-----------|---------|
| `transition` | `@include transition($transitions...)` | Prefixed `transition` |
| `transform` | `@include transform($transforms)` | Prefixed `transform` |
| `rotate` | `@include rotate($deg)` | Shorthand for `transform: rotate()` |
| `translate` | `@include translate($x, $y)` | Shorthand for `transform: translate()` |
| `scale` | `@include scale($scale)` | Shorthand for `transform: scale()` |
| `skew` | `@include skew($x, $y)` | Shorthand for `transform: skew()` |
| `animation` | `@include animation($animation)` | Prefixed `animation` |
| `gradient` | `@include gradient($direction, $list)` | Prefixed `linear-gradient` background |
| `background` | `@include background($background...)` | Unprefixed `linear-gradient` background |
| `background-image` | `@include background-image($deg, $colorname, $opacity)` | Unprefixed `linear-gradient` background |
| `x-appearance` | `@include x-appearance($value)` | Prefixed `appearance` |

**Usage:**

```scss
.my-element {
  @include transition(all 0.3s ease);
  @include transform(translateY(-50%));
}

.my-rotated {
  @include rotate(45);
}
```

### Placeholder selectors (from `_mixins.scss`)

Mangrove also defines `%placeholder` selectors for common patterns. Use these with `@extend`:

| Placeholder | Purpose |
|-------------|---------|
| `%paragraph-font-100` to `%paragraph-font-400` | Responsive font sizes (smaller on mobile, larger on desktop) |
| `%border-bottom` | `border-bottom: 1px solid $mg-color-neutral-400` |
| `%img-cover` | Full-width cover-fit image |
| `%left-to-right` | Entrance animation (translate + fade) |
| `%scale-up` | Scale entrance animation |
| `%opacity-only` | Fade-in animation |

## The `!default` override mechanism

The `!default` flag in SCSS means "use this value only if the variable has not already been assigned." This is how Mangrove supports theming.

In `_variables.scss`:

```scss
$mg-color-interactive: $mg-color-blue-900 !default;
$mg-radius-button: $mg-spacing-0 !default;
```

Mangrove's built-in themes import `_variables.scss` first, then a theme-specific override file that reassigns selected tokens **without** `!default`. Because the override comes second, its plain assignments overwrite the earlier `!default` values. Any token the override file does not touch keeps its default.

For example, `style-preventionweb.scss` imports the base variables first, then a theme-specific override file:

```scss
// style-preventionweb.scss
@import "./variables";              // Sets defaults
@import "./variables-preventionweb"; // Overrides the !default variables
@import "./fonts";
@import "./breakpoints";
@import "./mixins";
@import "./utility";
@import "./foundational";
@import "./components";
```

And `_variables-preventionweb.scss` reassigns the variables that need to differ:

```scss
$pw-color-primary: #0a6969;

$mg-color-interactive: $pw-color-primary;
$mg-color-interactive-active: $pw-color-primary;
$mg-color-button-background: $pw-color-primary;
$mg-color-button-background--hover: $sendai-orange-900;
$mg-radius-button: $mg-spacing-50;
// ...
```

Because these assignments happen after `_variables.scss` loads, they replace the `!default` values before any component stylesheet reads them.

## Creating a theme override

To create your own theme, write a variables override file and a theme entry point that follows the same import order.

### 1. Create your override file

```scss
// _variables-myorg.scss

// Define your brand color
$myorg-color-primary: #2e7d32;

// Override interactive colors
$mg-color-interactive: $myorg-color-primary;
$mg-color-interactive-active: darken($myorg-color-primary, 10%);

// Override button styling
$mg-color-button-background: $myorg-color-primary;
$mg-color-button-background--hover: $mg-color-interactive-active;
$mg-radius-button: $mg-spacing-50;

// Override tag colors
$mg-color-tag: $mg-color-interactive;
$mg-color-tag--hover: $mg-color-interactive-active;

// Override form colors
$mg-color-form-check: $mg-color-interactive;
$mg-color-form-focus: $mg-color-interactive;
$mg-radius-form-input: $mg-spacing-50;

// Override hero
$mg-color-hero: $mg-color-interactive;
$mg-spacing-hero-overlay: 2rem 0;
```

### 2. Create your theme entry point

```scss
// style-myorg.scss
@import "./variables";
@import "./variables-myorg";
@import "./fonts";
@import "./breakpoints";
@import "./mixins";
@import "./utility";
@import "./foundational";
@import "./components";
```

This follows the same structure as PreventionWeb, IRP, and MCR themes. All `!default` variables from `_variables.scss` that you reassigned in `_variables-myorg.scss` will carry through to every component stylesheet.

## Importing individual component styles

If you only need styles for specific components, import the partials directly. Each component has its own `.scss` file. The full list is in `stories/assets/scss/_components.scss`.

```scss
// Import base dependencies first
@import "stories/assets/scss/variables";
@import "stories/assets/scss/breakpoints";
@import "stories/assets/scss/mixins";
@import "stories/assets/scss/foundational";

// Then import only the components you need
@import "stories/Components/Buttons/CtaButton/buttons";
@import "stories/Components/Cards/Card/card";
@import "stories/Components/Forms/form-base";
@import "stories/Components/Tab/tab";
```

Note: many component stylesheets depend on variables and mixins from `_variables.scss`, `_breakpoints.scss`, and `_mixins.scss`, so always import those first.

## Build tool configuration

When using Mangrove's SCSS from `node_modules`, configure your Sass compiler to resolve imports from the package.

**Webpack:**

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          'style-loader',
          'css-loader',
          {
            loader: 'sass-loader',
            options: {
              sassOptions: {
                includePaths: [
                  path.resolve(__dirname, 'node_modules/@undrr/undrr-mangrove'),
                ],
              },
            },
          },
        ],
      },
    ],
  },
};
```

**Vite:**

```javascript
// vite.config.js
import { defineConfig } from 'vite';

export default defineConfig({
  css: {
    preprocessorOptions: {
      scss: {
        includePaths: [
          'node_modules/@undrr/undrr-mangrove',
        ],
      },
    },
  },
});
```

With `includePaths` configured, your imports resolve relative to the Mangrove package root:

```scss
@import "stories/assets/scss/variables";
@import "stories/assets/scss/breakpoints";
@import "stories/assets/scss/mixins";
```
