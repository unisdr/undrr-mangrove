# On this page nav

A sticky horizontal navigation bar that links to page sections, with scroll-spy highlighting and an optional call-to-action button. Vanilla JS — no React dependency.

**GitLab issue**: [web-backlog#337](https://gitlab.com/undrr/web-backlog/-/issues/337)

## When to use

Use **On this page nav** for long-form content pages (landing pages, reports, directories) where readers benefit from:
- Seeing the page structure at a glance
- Jumping quickly between sections
- Tracking where they are as they scroll

### On this page nav vs Table of contents

| | On this page nav | Table of contents |
|---|---|---|
| Position | Sticky at viewport top | Inline within content |
| Layout | Horizontal bar | Vertical list |
| Scroll-spy | Yes (highlights active section) | No |
| Best for | Persistent navigation on long pages | Static overview at the top of content |

## Modes

### Auto-detect (default)

Place an empty `<nav>` with the activation attribute. The script scans headings and generates links:

```html
<nav data-mg-on-this-page-nav
     data-mg-on-this-page-nav-content=".article-body"
     class="mg-on-this-page-nav">
</nav>
```

### Explicit

Provide your own `<ul>` with links. The script activates scroll-spy without scanning headings:

```html
<nav data-mg-on-this-page-nav class="mg-on-this-page-nav">
  <ul class="mg-on-this-page-nav__list">
    <li class="mg-on-this-page-nav__item">
      <a href="#intro" class="mg-on-this-page-nav__link">Introduction</a>
    </li>
    <li class="mg-on-this-page-nav__item">
      <a href="#methods" class="mg-on-this-page-nav__link">Methodology</a>
    </li>
  </ul>
</nav>
```

## Data attributes

| Attribute | Values | Default | Purpose |
|-----------|--------|---------|---------|
| `data-mg-on-this-page-nav` | (presence) | — | Activates the component |
| `data-mg-on-this-page-nav-depth` | `"2"`, `"3"`, `"4"` | `"2"` | Max heading level to scan (auto-detect mode) |
| `data-mg-on-this-page-nav-content` | CSS selector | `document.body` | Scope element for heading scan |
| `data-mg-on-this-page-nav-label` | string | `"On this page"` | `aria-label` for the nav element |
| `data-mg-on-this-page-nav-offset` | integer (px) | `"0"` | Scroll offset for fixed headers above the nav (e.g. MegaMenu height) |

## CTA button

Add a link with the `mg-on-this-page-nav__cta` class. It pins to the end of the bar (right in LTR, left in RTL):

```html
<nav data-mg-on-this-page-nav
     data-mg-on-this-page-nav-content=".article-body"
     class="mg-on-this-page-nav">
  <a href="/subscribe" class="mg-on-this-page-nav__cta">Subscribe to updates</a>
</nav>
```

In auto-detect mode, the CTA is preserved when the script generates links.

## Excluding headings

Add the `mg-on-this-page-nav--exclude` class to any heading to skip it:

```html
<h2 class="mg-on-this-page-nav--exclude">This heading won't appear in the nav</h2>
```

Headings with `mg-u-sr-only` are also excluded.

## JavaScript API

The component auto-initializes on `DOMContentLoaded` (or immediately if the script loads after the DOM is ready). For dynamic content or manual initialization:

```js
import { mgOnThisPageNav, mgOnThisPageNavDestroy } from 'path/to/on-this-page-nav.js';

// Initialize all [data-mg-on-this-page-nav] elements
mgOnThisPageNav();

// Initialize specific elements
mgOnThisPageNav([myNavElement]);

// Tear down (disconnects observer, removes init flag)
mgOnThisPageNavDestroy(myNavElement);
```

## Stacking with other sticky elements

The nav uses `z-index: 11`, which places it above the MegaMenu (10) but below the PageHeader (22) and modals (9998+). When both the MegaMenu and this nav are sticky on the same page, set `data-mg-on-this-page-nav-offset` to the MegaMenu's pixel height so headings scroll to the correct position.

## Accessibility

- **Landmark**: Renders as a `<nav>` with `aria-label`
- **Active section**: `aria-current="true"` on the active link (correct WCAG 1.3.1 pattern — not `aria-selected`)
- **List semantics**: `role="list"` on `<ul>` guards against VoiceOver + `list-style: none` bug
- **Keyboard**: Standard tab navigation through links; focused links auto-scroll into view in the nav bar
- **Focus management**: Clicking a link moves focus to the target heading (`tabindex="-1"`, removed on blur)
- **Focus visible**: 2px outline on `:focus-visible` for all interactive elements
- **Motion**: Smooth scroll and nav auto-scroll both respect `prefers-reduced-motion: reduce`
- **Zoom**: Un-sticks at `max-height: 400px` to prevent the bar from dominating the viewport at high zoom (per [GOV.UK accessibility research](https://technology.blog.gov.uk/2018/05/21/sticky-elements-functionality-and-accessibility-testing/))
- **Touch targets**: Links meet WCAG 2.5.8 Target Size (Minimum) at Level AA (24x24px). Vertical size is approximately 44px
- **Color contrast**: Active link color (`$mg-color-interactive`) passes 4.5:1 against white across all five themes

## RTL and internationalization

The component supports RTL layouts automatically:

- The nav list scrolls right-to-left via CSS flexbox and `overflow-x`
- The CTA button pins to the left (logical end) via `margin-inline-start: auto`
- Arabic headings use the Noto Kufi Arabic font family (`:lang(ar)` override)

Auto-generated IDs from non-Latin headings preserve the original characters. Provide explicit `id` attributes on headings if you need specific URL fragments.
