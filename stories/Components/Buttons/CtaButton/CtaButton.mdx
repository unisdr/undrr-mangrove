# Buttons

Mangrove's standard button component. Buttons are designed as call-to-action (CTA) elements — they should always direct the user toward a clear next step such as "Read more", "Sign up", or "View documentation."

#### When to use:

- Use buttons to drive a specific user action, not for navigation between pages (use links instead)
- Each page should have one primary action. Secondary and tertiary options should be used sparingly
- If there are multiple choices, one button should be the default (primary) and the others secondary or tertiary

#### Variants

| Class | Purpose |
|-------|---------|
| `.mg-button.mg-button-primary` | Main call to action — filled background, high contrast |
| `.mg-button.mg-button-secondary` | Supporting action — lower visual weight than primary |
| `.mg-button.disabled` | Inactive state — not clickable |

### Content

- Use clear, action-oriented labels: "Download report", "Sign up", not "Click here"
- Keep labels short — ideally 2-3 words

## Behaviors

Button colors adapt to the active theme (UNDRR, PreventionWeb, DELTA, etc.). In dark contexts like heroes and CTAs, buttons invert to ensure contrast.

### Usage

- Apply `.mg-button` as the base class, then add a variant class (`.mg-button-primary`, `.mg-button-secondary`)
- Copy the HTML from the HTML tab of the canvas

### Interactions

- Hover state applies to the entire button
