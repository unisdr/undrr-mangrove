# Buttons and labels

A button describes what action will be performed when a user clicks or touches it.

### Overview

A button describes the call to action that will be performed when the user clicks/touches it. It has a label and an optional animation.

#### When to use:

* Buttons should be used as primary, secondary or tertiary calls to actions
* Ideally, a page should have one primary action. A secondary/tertiary option should be used infrequently.
* If there are multiple choices for an action item, one default button should be active and the other buttons inactive

### Formatting

#### Default

There are multiple button types in our design system

1. Primary/Secondary/tertiary buttons
2. CTA links with and without expanding arrows
3. Chips

### Content

Buttons, Chips and CTA link colors and text indicate the default setting and call of action of the button.

## Behaviors

The default colors of the buttons will depend on the theme in use such as PW, UNDRR etc.
