# Select

A native dropdown select for choosing from a list of options.

## Overview

The select component provides a native `<select>` dropdown with consistent styling, label support, and accessible error/help states. It uses the browser's native select behavior for maximum compatibility.

## When to use

- When the user needs to choose a single option from a predefined list.
- For lists of 4 or more options (for fewer options, consider radio buttons).

## States

- **Default**: Ready for selection.
- **Focus**: Blue outline on keyboard focus.
- **Disabled**: Greyed out, non-interactive.
- **Error**: Red border with an error message below.

## Accessibility

- The label is programmatically associated with the select via `htmlFor`/`id`.
- Help text and error text are linked via `aria-describedby`.
- Error state sets `aria-invalid="true"` and wraps the error message in `role="alert"`.
- Uses native `<select>` for full keyboard and screen reader support.
