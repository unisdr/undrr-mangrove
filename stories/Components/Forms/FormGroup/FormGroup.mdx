# Form group

An accessible wrapper for grouping related form controls using `<fieldset>` and `<legend>`.

## Overview

The FormGroup component renders a native `<fieldset>` with a `<legend>`, which satisfies WCAG 1.3.1 (Info and Relationships). Screen readers announce the legend text before each control in the group, so users know how the options relate.

## When to use

- **Always** wrap related checkboxes in a FormGroup so users know what the set of choices represents.
- **Always** wrap related radio buttons in a FormGroup so users understand the question being asked.
- When you need to disable or show an error on an entire group of controls at once.

## Hidden legend

Use `hideLegend` when the group's purpose is visually obvious from context but still needs an accessible label. The legend is visually hidden but remains in the accessibility tree.

## Accessibility

- Renders a native `<fieldset>` with `<legend>` per [WAI-ARIA APG](https://www.w3.org/WAI/ARIA/apg/practices/names-and-descriptions/).
- Screen readers announce the legend before each control in the group.
- The `disabled` attribute on `<fieldset>` natively disables all descendant controls.
- Error messages use `role="alert"` for live announcements.
- `hideLegend` keeps the legend accessible to screen readers while hiding it visually.
