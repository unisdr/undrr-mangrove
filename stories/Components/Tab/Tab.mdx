# Tabs

Tabs let users switch between related groups of content without leaving the page. One panel is visible at a time.

### When to use

- Organizing content into sections a user can flip between (e.g. overview, details, resources).
- The content groups are peers, and users don't need to see them side by side.

Don't use tabs for site navigation. Use links or menus instead.

### Variants

There are two layouts. **Horizontal** is a tab bar along the top, best for 2ÔÇô5 tabs with short labels. **Stacked** is a vertical, accordion-style layout that works better for longer labels or many sections. On screens Ôëñ600 px, horizontal tabs collapse to stacked automatically.

### Interactions

**Horizontal tabs:**
- Hovering highlights the full tab.
- Clicking a tab opens its panel.
- Arrow Left/Right moves between tabs. Down arrow moves focus into the open panel. Home/End jump to first/last tab.

**Stacked tabs:**
- Clicking a trigger toggles its panel open/closed independently.
- Arrow Up/Down moves focus between triggers without toggling. Home/End jump to first/last trigger.
- With `singleOpen`, opening one panel closes all others (accordion behavior).

URL hashes work as deep links in both variants, e.g. `#mg-tabs__section-tab-2` opens that section on load.

## Usage

Tabs are vanilla JavaScript. No React, no build step, just HTML and a script import. If you're already working in React with the Mangrove source, there's also a `<Tab>` wrapper ([see below](#react-usage)).

### Vanilla JS (recommended)

#### 1. Include the CSS

Pick the Mangrove theme stylesheet for your site ([CDN reference](/?path=/docs/getting-started-integration-cdn-reference--docs) lists all themes).

```html
<link rel="stylesheet" href="https://assets.undrr.org/static/mangrove/1.3.3/css/style.css" />
```

#### 2. Include the JavaScript

Import `mgTabs` from the CDN and call it once the DOM is ready:

```html
<script type="module">
  import { mgTabs } from 'https://assets.undrr.org/static/mangrove/1.3.3/js/tabs.js';
  mgTabs();
</script>
```

Module scripts are deferred by default, so the DOM is already parsed when this runs. No `DOMContentLoaded` wrapper needed.

`mgTabs()` finds every `[data-mg-js-tabs]` element on the page and wires up the tab behavior.

#### 3. Add the HTML markup

**Horizontal tabs (default)**

```html
<article class="mg-tabs mg-tabs--horizontal"
         data-mg-js-tabs
         data-mg-js-tabs-variant="horizontal">
  <ul class="mg-tabs__list">
    <!-- Tab 1 -->
    <li class="mg-tabs__item">
      <a class="mg-tabs__link" href="#mg-tabs__section-tab-1">Tab title 1</a>
    </li>
    <li class="mg-tabs-content" data-mg-js-tabs-content>
      <section class="mg-tabs__section" id="mg-tabs__section-tab-1">
        <p>Content for tab 1.</p>
      </section>
    </li>

    <!-- Tab 2 (default selected) -->
    <li class="mg-tabs__item">
      <a class="mg-tabs__link" href="#mg-tabs__section-tab-2"
         data-mg-js-tabs-default="true">Tab title 2</a>
    </li>
    <li class="mg-tabs-content" data-mg-js-tabs-content>
      <section class="mg-tabs__section" id="mg-tabs__section-tab-2">
        <p>Content for tab 2.</p>
      </section>
    </li>

    <!-- Tab 3 -->
    <li class="mg-tabs__item">
      <a class="mg-tabs__link" href="#mg-tabs__section-tab-3">Tab title 3</a>
    </li>
    <li class="mg-tabs-content" data-mg-js-tabs-content>
      <section class="mg-tabs__section" id="mg-tabs__section-tab-3">
        <p>Content for tab 3.</p>
      </section>
    </li>
  </ul>
</article>
```

**Stacked tabs (accordion-style)**

Change the modifier class and variant attribute:

```html
<article class="mg-tabs mg-tabs--stacked"
         data-mg-js-tabs
         data-mg-js-tabs-variant="stacked">
  <ul class="mg-tabs__list">
    <li class="mg-tabs__item">
      <a class="mg-tabs__link" href="#mg-tabs__section-item-1">Section 1</a>
    </li>
    <li class="mg-tabs-content" data-mg-js-tabs-content>
      <section class="mg-tabs__section" id="mg-tabs__section-item-1">
        <p>Content for section 1.</p>
      </section>
    </li>

    <li class="mg-tabs__item">
      <a class="mg-tabs__link" href="#mg-tabs__section-item-2">Section 2</a>
    </li>
    <li class="mg-tabs-content" data-mg-js-tabs-content>
      <section class="mg-tabs__section" id="mg-tabs__section-item-2">
        <p>Content for section 2.</p>
      </section>
    </li>
  </ul>
</article>
```

**Stacked tabs as FAQ (all closed, single-open)**

Add `data-mg-js-tabs-default-open="false"` to start all panels closed, and `data-mg-js-tabs-single-open` for accordion behavior:

```html
<article class="mg-tabs mg-tabs--stacked"
         data-mg-js-tabs
         data-mg-js-tabs-variant="stacked"
         data-mg-js-tabs-default-open="false"
         data-mg-js-tabs-single-open>
  <ul class="mg-tabs__list">
    <!-- same item structure as above -->
  </ul>
</article>
```

#### API

Exported functions:

| Function | Description |
|----------|-------------|
| `mgTabs(scope?, activateDeepLinkOnLoad?)` | Initializes all tab containers. `scope` is an optional `NodeList`; defaults to all `[data-mg-js-tabs]` elements. `activateDeepLinkOnLoad` defaults to `true`. |
| `mgTabsRuntime(scope?, activateDeepLinkOnLoad?)` | Initializes tabs within a scope element. `scope` is an optional DOM element (a tab container or a parent that contains them); defaults to `document`. Use this for tabs injected after page load. |
| `mgTabsApplyStackedDefaults(container, tabs, panels)` | Re-applies default open/close state for stacked tabs. Useful after programmatically resetting filter state. |
| `setDisclosureState(trigger, panel, open)` | Sets the open/close state of a stacked disclosure panel. `open` is a boolean. |

If you inject tab markup dynamically (e.g. from an API response), call `mgTabsRuntime()` on the new container:

```js
const newContainer = document.querySelector('#dynamic-tabs');
mgTabsRuntime(newContainer);
```

#### Key data attributes

| Attribute | Element | Purpose |
|-----------|---------|---------|
| `data-mg-js-tabs` | `article` | Identifies a tab container |
| `data-mg-js-tabs-variant` | `article` | `"horizontal"` or `"stacked"` |
| `data-mg-js-tabs-content` | `li` | Wraps each tab's content section |
| `data-mg-js-tabs-default` | `a.mg-tabs__link` | Set to `"true"` to make this tab active by default (per-item override) |
| `data-mg-js-tabs-default-open` | `article` | `"true"` opens all stacked panels on load, `"false"` closes all. Absent = open first. Per-item `data-mg-js-tabs-default` overrides this. |
| `data-mg-js-tabs-single-open` | `article` | Presence-based. When set, only one stacked panel can be open at a time (accordion). |

### React usage

The `<Tab>` component renders the same HTML as above and calls `mgTabs()` on mount. It's available from the Mangrove source but not published as a precompiled bundle on the CDN.

```jsx
// Requires the full Mangrove source tree (not available from the npm package alone)
import { Tab } from 'stories/Components/Tab/Tab';

const tabdata = [
  { text: 'Overview', text_id: 'overview', data: '<p>Overview content</p>' },
  { text: 'Details', text_id: 'details', data: '<p>Detail content</p>', is_default: 'true' },
];

<Tab tabdata={tabdata} variant="horizontal" />
```

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tabdata` | `Array` | ÔÇö | Array of tab objects (see below) |
| `variant` | `string` | `"horizontal"` | `"horizontal"` or `"stacked"` |
| `defaultOpen` | `bool` | `undefined` | Stacked only. `true` opens all panels, `false` closes all, `undefined` opens first. Per-item `is_default` overrides. |
| `singleOpen` | `bool` | `false` | Stacked only. When `true`, only one panel can be open at a time. |
| `filterable` | `bool` | `false` | Stacked only. Renders a search input that filters sections by header and content text. |
| `filterPlaceholder` | `string` | `"Filter sectionsÔÇª"` | Placeholder and `aria-label` for the filter input. |

Each tab object in `tabdata`:

| Property | Type | Description |
|----------|------|-------------|
| `text` | `string` | Tab label |
| `text_id` | `string` | Unique identifier (used in `id` and `href` attributes) |
| `data` | `string` | HTML content for the tab panel |
| `is_default` | `string` | Set to `"true"` to select this tab on load |
