# Quote highlight

The `QuoteHighlight` component is used to display important quotes with attribution and optional images.

## Features

- Two layout variants: line (with separator line) or image (with side image)
- Three background color options: light, dark, and bright
- Optional attribution with name and title
- Optional portrait image that displays inline with attribution
- Optional side image for the image variant
- Three alignment options: full, left, and right
- RTL (Right-to-Left) language support
- Responsive design that adapts to different screen sizes

## Usage

```jsx
import { QuoteHighlight } from '@undrr/undrr-mangrove';

<QuoteHighlight
  quote="One doesn't have to look hard to find examples of how disasters are becoming worse."
  attribution="Mami Mizutori"
  attributionTitle="Special Representative of the UN Secretary-General for Disaster Risk Reduction"
  imageSrc="/assets/author.png"
  imageAlt="Mami Mizutori portrait"
  variant="line"
  backgroundColor="light"
  alignment="full"
/>;
```

## Alignment options

The QuoteHighlight component supports three alignment options:

- `full`: Takes up the full width of its container (default)
- `left`: Aligns to the left, allowing content to flow around the right side
- `right`: Aligns to the right, allowing content to flow around the left side

## RTL support

The QuoteHighlight component fully supports Right-to-Left (RTL) languages. When used in an RTL context:

- Left-aligned quotes will appear on the right side of the content
- Right-aligned quotes will appear on the left side of the content
- The component's internal layout will automatically adjust for RTL reading direction

## Design guidelines

- Use the QuoteHighlight component to emphasize important quotes from key stakeholders.
- The line variant is more subtle and can be used within text content.
- The image variant is more prominent and works well as a standalone element.
- Choose background colors that complement the surrounding content.
- Always provide attribution when available to give proper credit to the source.
- Use the portrait image feature to add a personal touch to the quote.
- Ensure images are high quality and properly cropped to fit the designated space.
- Use alignment options to create more dynamic layouts:
  - Full width for standalone, prominent quotes
  - Left or right alignment to integrate quotes within text content
- For RTL languages, the component will automatically mirror its layout.

## Accessibility

### Semantic HTML

- The component uses a `<blockquote>` element for the quote text, giving screen readers proper quote semantics
- The outer wrapper is a `<section>` element, providing a sectioning landmark
- Attribution name and title are rendered as `<p>` elements within the attribution area

### Image alt text

- The `imageAlt` prop controls the `alt` attribute on both the portrait image (line variant) and the side image (image variant)
- When `imageAlt` is not provided, the component generates a fallback: `"{attribution} image"`, or `"Quote image"` if no attribution is set
- Always provide a meaningful `imageAlt` value rather than relying on the fallback, especially when the image conveys information beyond decoration

### Known limitations

- The `<section>` element has no `aria-label` or `aria-labelledby`, so screen readers may announce it as an unlabelled section
- No `<figure>` / `<figcaption>` pattern is used; the quote and its attribution are not programmatically associated (e.g., via `aria-describedby` or `<footer>` within the `<blockquote>`)
- The `<cite>` element is not used for the attribution, which would give screen readers stronger semantic context for the source of the quote
- The fallback alt text (e.g., "Mami Mizutori image") may not be descriptive enough for images that convey meaningful content

## Server-rendered / CDN usage

QuoteHighlight supports [layered hydration](https://github.com/unisdr/undrr-mangrove/blob/main/docs/HYDRATION.md). Use data attributes on a container element:

```html
<div data-mg-quote-highlight
  data-quote="One doesn't have to look hard to find examples."
  data-attribution="Mami Mizutori"
  data-attribution-title="SRSG for Disaster Risk Reduction"
  data-image-src="/assets/author.png"
  data-image-alt="Mami Mizutori portrait"
  data-background-color="light"
  data-variant="line"
  data-alignment="full">
</div>
```

| Attribute | Default | Description |
|-----------|---------|-------------|
| `data-quote` | `''` | Quote text (required) |
| `data-attribution` | — | Attribution name |
| `data-attribution-title` | — | Attribution title or role |
| `data-image-src` | — | Portrait or side image URL |
| `data-image-alt` | — | Alt text for the image |
| `data-background-color` | `'light'` | `'light'`, `'dark'`, or `'bright'` |
| `data-variant` | `'line'` | `'line'` or `'image'` |
| `data-alignment` | `'full'` | `'full'`, `'left'`, or `'right'` |

```js
import createHydrator from "https://assets.undrr.org/static/mangrove/1.4.0/components/hydrate.js";
import QuoteHighlight, { fromElement } from "https://assets.undrr.org/static/mangrove/1.4.0/components/QuoteHighlight.js";
createHydrator({ selector: "[data-mg-quote-highlight]", component: QuoteHighlight, fromElement });
```

See the [hydration guide](https://github.com/unisdr/undrr-mangrove/blob/main/docs/HYDRATION.md) for the full API and integration details.
