# Vertical card

A content teaser card in portrait/stacked layout. Image on top, text below. Used across all UNDRR sites for news, events, publications, and general content listings.

## When to use

- Content listings and search results in grid layouts
- News and event teasers
- Any content that benefits from a prominent image above a title and summary

## Anatomy

- **Image** (optional): 16:9 aspect ratio, full width of the card
- **Labels** (optional): Content type or category tags
- **Title**: Linked heading with blue color, underline, and chevron indicator
- **Summary** (optional): Body text, supports inline HTML links
- **CTA button** (optional): Primary action button

The entire card is clickable via a stretched link on the title.

## Title link visibility

By default, card titles display three signals that they are clickable:

1. **Blue color** matching `$mg-color-interactive` (adapts per theme)
2. **Underline** with `3px` offset
3. **Chevron indicator** (CSS border triangle via `::after`)

On hover, the underline thickens to `2px`. In RTL layouts, the chevron flips direction automatically.

### Opting out

To disable link visibility signals (for example, on cards where the title should look like a plain heading), add the `mg-card--plain-title` class.

**In React:**

```jsx
<VerticalCard data={data} className="mg-card--plain-title" />
```

**In Drupal templates or static HTML:**

```html
<article class="mg-card mg-card__vc mg-card--plain-title">
  ...
</article>
```

This removes the blue color, underline, and chevron. The card remains clickable via the stretched link.

## Accessibility

- The stretched link (`::before` on the title `<a>`) makes the entire card clickable
- Inline links in the summary use `z-index: 101` to remain independently clickable above the stretched link
- Focus states are visible on all interactive elements
- RTL layout is automatic when `dir="rtl"` is set on the document
