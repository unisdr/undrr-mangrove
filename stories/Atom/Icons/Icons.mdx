# Icons

The Icon components grab attention and draw users into content, helping them quickly understand the context in which they are placed.

Icons use the **Mangrove** namespace (`mg-icon-*`) as the preferred format. Legacy `fa-*` (FontAwesome-style) classes remain supported for backwards compatibility.

## When to use:

- **Dense and Complex Content**: Use icons to help users comprehend available actions in complex interfaces.
- **Space Saving**: Replace words or long phrases with icons to save space without losing meaning.
- **Avoid Visual Clutter**: Use icons sparingly to prevent overwhelming the user with too many visual elements.

## Usage

### React Component (Recommended)

```jsx
import { Icon } from './Icon';

// Just the icon name (recommended)
<Icon name="globe" />

// With size modifier
<Icon name="globe" size="lg" />

// Legacy fa- format (still supported)
<Icon name="fa-globe" />
```

### CSS Classes

Use `mg-icon` base class with `mg-icon-{name}` (preferred) or `fa-{name}` (legacy):

```html
<!-- Preferred format -->
<span class="mg-icon mg-icon-globe"></span>
<span class="mg-icon mg-icon-chart-bar"></span>

<!-- Legacy format (still supported) -->
<span class="mg-icon fa-globe"></span>
<span class="mg-icon fa-rss"></span>
```

### Size Modifiers

```html
<span class="mg-icon mg-icon-globe mg-icon--sm"></span>  <!-- 0.875em -->
<span class="mg-icon mg-icon-globe mg-icon--lg"></span>  <!-- 1.5em -->
<span class="mg-icon mg-icon-globe mg-icon--xl"></span>  <!-- 2em -->
```
