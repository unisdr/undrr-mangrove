# Icons

Icons help users quickly understand context and available actions. They render via CSS `mask-image` backed by SVG data URIs, so the CSS `color` property controls the icon color.

Icons come from three sources, shown with colored badges in the gallery below:

- **Lucide** (green) — general UI icons (search, menu, arrows, etc.). [lucide.dev](https://lucide.dev), MIT license.
- **OCHA** (blue) — humanitarian and disaster risk reduction icons (earthquake, flood, cyclone, etc.). [OCHA Humanitarian Icons](https://github.com/UN-OCHA/humanitarian-icons), CC0-1.0 license.
- **Custom** (orange) — brand logos and UNDRR-specific icons maintained in `stories/assets/icons/custom/`.

Some icons have aliases for backward compatibility (e.g., `close` maps to `times`, `copy` to `clone`, `twitter` to `x-social`). Aliases appear in the gallery and work identically to the canonical name. See `scripts/icon-map.cjs` for the full alias list.

## When to use

- **Dense content**: Icons help users scan available actions in complex interfaces.
- **Space saving**: Replace words or long phrases with icons where meaning is clear.
- **Use sparingly**: Too many icons create visual noise instead of reducing it.

## Usage

### React component (recommended)

```jsx
import { Icon } from '@undrr/undrr-mangrove';

// Just the icon name (recommended)
<Icon name="globe" />

// With size modifier
<Icon name="globe" size="lg" />

// With color utility or custom class
<Icon name="globe" className="mg-u-color--interactive" />

// Multi-color (preserves SVG's original colors)
<Icon name="sdg-wheel" className="mg-icon--multicolor" />

// Legacy fa- format (still supported via font fallback)
<Icon name="fa-globe" />
```

### CSS classes

Use `mg-icon` base class with `mg-icon-{name}`:

```html
<span class="mg-icon mg-icon-globe"></span>
<span class="mg-icon mg-icon-chart-bar"></span>
<span class="mg-icon mg-icon-earthquake"></span>
```

Legacy `fa-*` classes still work via the icon font fallback. New code should use `mg-icon-*`.

### Size modifiers

```html
<span class="mg-icon mg-icon-globe mg-icon--sm"></span>  <!-- 0.875em -->
<span class="mg-icon mg-icon-globe mg-icon--lg"></span>  <!-- 1.5em -->
<span class="mg-icon mg-icon-globe mg-icon--xl"></span>  <!-- 2em -->
```

### Coloring

Icons use `currentColor`, so they respond to any method of setting `color`:

**Color utility classes** — the existing `mg-u-color--*` utilities work directly on icons:

```html
<span class="mg-icon mg-icon-globe mg-u-color--interactive"></span>
<span class="mg-icon mg-icon-times mg-u-color--red-800"></span>
<span class="mg-icon mg-icon-power-off mg-u-color--white"></span>
```

**Parent inheritance** — icons inside a colored element inherit that color:

```html
<div class="mg-u-color--interactive">
  <span class="mg-icon mg-icon-globe"></span>
  <span class="mg-icon mg-icon-search"></span>
</div>
```

**Inline style** — for one-off colors (e.g. brand logos):

```html
<span class="mg-icon mg-icon-facebook" style="color: #1877f2"></span>
```

**Multi-color** — by default icons are single-color (the mask fills everything with `currentColor`). Add `mg-icon--multicolor` to render the SVG with its original fill/stroke colors preserved:

```html
<!-- Single-color (default) — SVG rendered as currentColor silhouette -->
<span class="mg-icon mg-icon-sdg-wheel"></span>

<!-- Multi-color — SVG rendered with its original colors -->
<span class="mg-icon mg-icon-sdg-wheel mg-icon--multicolor"></span>
```

Multi-color only works with SVGs that have explicit fill/stroke color values in the source file (like the SDG wheel or brand logos). Lucide and OCHA icons use `currentColor` in their source SVGs, so applying `mg-icon--multicolor` to them won't produce a visible difference.

### Migrating from `fa-*` to `mg-icon-*`

For most icons, replace `fa-` with `mg-icon-`:

| Old class | New class |
|---|---|
| `fa-globe` | `mg-icon-globe` |
| `fa-search` | `mg-icon-search` |
| `fa-envelope` | `mg-icon-envelope` |
| `fa-times` | `mg-icon-times` (or `mg-icon-close`) |
| `fa-bar-chart` | `mg-icon-chart-bar` (or `mg-icon-bar-chart`) |
| `fa-file-alt` | `mg-icon-file-alt` |
| `fa-calendar-alt` | `mg-icon-calendar-alt` (or `mg-icon-calendar`) |
| `fa-clone` | `mg-icon-clone` (or `mg-icon-copy`) |
| `fa-twitter` | `mg-icon-x-social` (or `mg-icon-twitter`) |
| `fa-left` | `mg-icon-left` (or `mg-icon-arrow-left`) |
| `fa-right` | `mg-icon-right` (or `mg-icon-arrow-right`) |
| `fa-menu` | `mg-icon-menu` (or `mg-icon-hamburger`) |

The full list is in `scripts/icon-map.cjs`. Legacy `fa-*` classes still work during the transition but will be removed in a future release (see [#906](https://github.com/unisdr/undrr-mangrove/issues/906) phases 3-4).

## Adding a new icon

1. **Find or create the SVG.** Browse [Lucide](https://lucide.dev) for UI icons, [OCHA Humanitarian Icons](https://github.com/UN-OCHA/humanitarian-icons) for domain icons, or create a custom SVG. Custom SVGs must use a `24 24` viewBox. Place custom files in `stories/assets/icons/custom/`.

2. **Add an entry to `scripts/icon-map.cjs`:**
   ```js
   // For a Lucide icon:
   'my-icon': LUCIDE('lucide-icon-name'),
   // For a custom SVG at stories/assets/icons/custom/my-icon.svg:
   'my-icon': CUSTOM('my-icon'),
   // For an OCHA icon at stories/assets/icons/ocha/my-icon.svg:
   'my-icon': OCHA('my-icon'),
   ```

3. **Add the icon to `Icons.json`** with `name`, `label`, and `source` fields. This drives the Storybook gallery — icons not in this file won't appear in the gallery but will still work in CSS.

4. **Run `yarn build:icons`** to regenerate `_icon-definitions.scss`. The generated file is already imported by `icons.scss`, so no manual import step is needed.

5. The icon is now available as `<Icon name="my-icon" />` and `<span class="mg-icon mg-icon-my-icon"></span>`.

## Technical notes

### How it works

Each icon definition sets a `--mg-icon-svg` CSS custom property containing the SVG as a data URI. The base `.mg-icon[class*="mg-icon-"]::before` rule references it via `mask-image: var(--mg-icon-svg)`, so each data URI is defined once rather than duplicated.

The `mg-icon--multicolor` modifier switches from `mask-image` (single-color silhouette) to `background-image: var(--mg-icon-svg)` (original SVG colors preserved).

### Architecture

- Icon definitions are auto-generated by `scripts/icon-build.cjs` from the mapping in `scripts/icon-map.cjs`. Run `yarn build:icons` to regenerate.
- Definitions compile into a standalone `mangrove-icons.css` file, shared across all themes (not duplicated per theme).
- The icon font (`mangrove-icon-set`) is still loaded as a fallback for `fa-*` selectors. It will be removed in a future release once all consumers migrate to `mg-icon-*` (see [#906](https://github.com/unisdr/undrr-mangrove/issues/906) phases 3-4).
- Windows High Contrast Mode: `forced-color-adjust: none` prevents the browser from overriding mask colors, and a `@media (forced-colors: active)` rule sets `background-color: CanvasText` so icons remain visible.
- `mask-image` requires Chrome 120+, Firefox 53+, Safari 15.4+, Edge 120+. CSS custom properties (`var()`) have wider support, so they add no additional browser requirement. See [browser support](https://unisdr.github.io/undrr-mangrove/?path=/docs/getting-started-browser-support--docs) for the full baseline.

## Accessibility

- Icons use `aria-hidden="true"` since they are decorative. If an icon conveys meaning on its own, pair it with visually hidden text.
- Icons remain visible in Windows High Contrast Mode (forced-colors).
- Icons scale with browser zoom via `em` units.
- When an icon is the sole content of a button or link, the interactive element must meet a minimum target size of 24x24 CSS pixels (WCAG 2.5.8). The `mg-share__button` class (40x40px) is a good reference pattern.
