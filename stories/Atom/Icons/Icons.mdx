# Icons

Icons help users quickly understand context and available actions. They render via CSS `mask-image` backed by SVG data URIs, so the CSS `color` property controls the icon color.

Icons come from three sources, shown with colored badges in the gallery below:

- **Lucide** (green) — general UI icons (search, menu, arrows, etc.). [lucide.dev](https://lucide.dev), MIT license.
- **OCHA** (blue) — humanitarian and disaster risk reduction icons (earthquake, flood, cyclone, etc.). [OCHA Humanitarian Icons](https://github.com/UN-OCHA/humanitarian-icons), CC0-1.0 license.
- **Custom** (orange) — brand logos and UNDRR-specific icons maintained in `stories/assets/icons/custom/`.

## When to use

- **Dense content**: Icons help users scan available actions in complex interfaces.
- **Space saving**: Replace words or long phrases with icons where meaning is clear.
- **Use sparingly**: Too many icons create visual noise instead of reducing it.

## Usage

### React component (recommended)

```jsx
import { Icon } from './Icon';

// Just the icon name (recommended)
<Icon name="globe" />

// With size modifier
<Icon name="globe" size="lg" />

// Legacy fa- format (still supported via font fallback)
<Icon name="fa-globe" />
```

### CSS classes

Use `mg-icon` base class with `mg-icon-{name}`:

```html
<span class="mg-icon mg-icon-globe"></span>
<span class="mg-icon mg-icon-chart-bar"></span>
<span class="mg-icon mg-icon-earthquake"></span>
```

Legacy `fa-*` classes still work via the icon font fallback. New code should use `mg-icon-*`.

### Size modifiers

```html
<span class="mg-icon mg-icon-globe mg-icon--sm"></span>  <!-- 0.875em -->
<span class="mg-icon mg-icon-globe mg-icon--lg"></span>  <!-- 1.5em -->
<span class="mg-icon mg-icon-globe mg-icon--xl"></span>  <!-- 2em -->
```

### Coloring

Icons use `currentColor`, so they respond to any method of setting `color`:

**Color utility classes** — the existing `mg-u-color--*` utilities work directly on icons:

```html
<span class="mg-icon mg-icon-globe mg-u-color--interactive"></span>
<span class="mg-icon mg-icon-times mg-u-color--red-800"></span>
<span class="mg-icon mg-icon-power-off mg-u-color--white"></span>
```

**Parent inheritance** — icons inside a colored element inherit that color:

```html
<div class="mg-u-color--interactive">
  <span class="mg-icon mg-icon-globe"></span>
  <span class="mg-icon mg-icon-search"></span>
</div>
```

**Inline style** — for one-off colors (e.g. brand logos):

```html
<span class="mg-icon mg-icon-facebook" style="color: #1877f2"></span>
```

## Technical notes

- Icons render via CSS `mask-image` on `::before`. The SVG acts as a stencil; `background-color: currentColor` fills the visible area.
- Icon definitions are auto-generated by `scripts/icon-build.cjs` from the mapping in `scripts/icon-map.cjs`. Run `yarn build:icons` to regenerate.
- Definitions compile into a standalone `mangrove-icons.css` file, shared across all themes (not duplicated per theme).
- The icon font (`mangrove-icon-set`) is still loaded as a fallback for `fa-*` selectors. It will be removed in a future release once all consumers migrate to `mg-icon-*`.
- Windows High Contrast Mode is handled via `forced-color-adjust: none` and a `@media (forced-colors: active)` fallback that sets `background-color: CanvasText`.

### Adding a new icon

1. Find the SVG in [Lucide](https://lucide.dev), [OCHA Humanitarian Icons](https://github.com/UN-OCHA/humanitarian-icons), or create a custom 24x24 SVG.
2. Add an entry to `scripts/icon-map.cjs` mapping the `mg-icon-*` name to the SVG path.
3. Add the icon to `Icons.json` with a `name`, `label`, and `source` field.
4. Run `yarn build:icons` to regenerate `_icon-definitions.scss`.
5. The icon is now available as `<Icon name="your-icon" />` and `<span class="mg-icon mg-icon-your-icon"></span>`.

## Accessibility

- Icons use `aria-hidden="true"` since they are decorative. If an icon conveys meaning on its own, pair it with visually hidden text.
- Icons remain visible in Windows High Contrast Mode (forced-colors).
- Icons scale with browser zoom via `em` units.
