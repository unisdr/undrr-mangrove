# Scripts

This directory contains utility scripts for the Mangrove project.

## update-cdn-version.js

Automatically updates all CDN links in documentation files to use the current version from `package.json` instead of "latest" or "testing" endpoints.

### Usage

```bash
# Run the script directly
node scripts/update-cdn-version.js

# Or use the npm script
yarn update-cdn-version
```

### What it does

1. Reads the current version from `package.json`
2. Updates all documentation files to replace:
   - `https://assets.undrr.org/testing/static/mangrove/latest/` → `https://assets.undrr.org/static/mangrove/{version}/`
   - `https://assets.undrr.org/static/mangrove/latest/` → `https://assets.undrr.org/static/mangrove/{version}/`

### Files updated

- `README.md`
- `stories/Documentation/GettingStarted.mdx`
- `stories/Documentation/VanillaHtmlCss.mdx`
- `stories/Documentation/Intro.mdx`
- `stories/Components/StatsCardSlider/StatsCardSlider.mdx`
- `stories/Utilities/ShowMore/ShowMore.mdx`
- `stories/Atom/Layout/Grid/Grid.mdx`
- `docs/RELEASES.md`

### When to use

Run this script after each release to ensure all documentation points to the stable, versioned CDN assets instead of bleeding-edge ones.

## generate-ai-manifest.js

Generates machine-readable component metadata for AI coding agents. Runs automatically during `yarn build` (after `storybook build`, before `webpack`).

### Usage

```bash
# Run the script directly (requires a prior storybook build)
node scripts/generate-ai-manifest.js

# Or use the npm script
yarn generate-ai-manifest

# Custom build output directory
node scripts/generate-ai-manifest.js --build-dir=docs-build-temp
```

### What it generates

The script reads Storybook's internal components manifest (`docs-build-temp/manifests/components.json`, produced by the `experimentalComponentsManifest` feature) and transforms it into three tiers of AI-friendly files:

| File | Size | Purpose |
|------|------|---------|
| `llms.txt` | ~1 KB | Plain-text discovery file ([llmstxt.org](https://llmstxt.org/) convention). Project overview, key conventions, pointer to the component index. |
| `ai-components/index.json` | ~29 KB | Lightweight index of all components with names, descriptions, import statements, and `detailsUrl` links to per-component files. |
| `ai-components/{id}.json` | 1-5 KB each | Full component metadata: props with types/defaults/descriptions, and code examples from Storybook stories. |

### How it works

1. Storybook's `experimentalComponentsManifest` feature extracts component metadata via `react-docgen` during `storybook build`
2. This script reads that manifest (~700 KB, internal format) and transforms it into the tiered structure above
3. The output files are written to the Storybook build directory (`docs-build-temp/`) and deploy alongside the static site to GitHub Pages
4. AI agents discover the manifest via `llms.txt` at the site root or by fetching `ai-components/index.json` directly

### Data sources

Component metadata is extracted from:

- **Props**: `PropTypes` definitions and JSDoc `@param` tags in component `.jsx` files
- **Descriptions**: JSDoc comments on component functions
- **Code examples**: Storybook story render functions (CSF3 format)
- **Import statements**: Generated by Storybook from `src/index.js` exports

To improve the AI manifest quality, add JSDoc descriptions to components and use descriptive PropTypes comments.
